# examples/ — 參照小說片段（對照分析用）

這裡放的是一批**公認好讀**的網路小說片段，用途是：先分析它們「好在哪」（具體 vs 抽象敘事、設定如何被自然帶出、資訊揭露節奏、句子可讀性…），再回頭對照本 repo 用 `write` skill 產出的原創故事「芯片巫師」，找出讀起來「碎片感、生硬、設定突然蹦出」的**具體原因**。

作者刻意選了**非開頭**或**開頭＋前段**的片段——開頭為了吸引讀者節奏特殊，不代表常態敘事。

> ⚠️ **第三方版權內容**：以下正文均為各作者著作，僅供本專案**個人寫作研究/風格對照**之用，非再散布。請勿外流或商用。

## 目錄結構

```
examples/
  download.py          # 下載腳本（PEP 723，uv run 直接跑）
  README.md            # 本檔
  <書名>/
    manifest.json      # title/author/source/id/range/fetched + 每章 {number,title,cid,file,url}
    第001章_<章名>.txt  # 一章一檔：首行章名、空行、之後每段一行純正文
    ...
```

## 用法

於 **repo 根目錄**執行（環境用 uv）：

```bash
uv run examples/download.py --list           # dry-run：只印「範圍→章號/章名/檔名」對映，不下載
uv run examples/download.py                  # 下載全部六本
uv run examples/download.py --only 從紅月開始  # 只下載一本
```

腳本禮貌設定：每章間隔 1.5s、失敗重試 3 次；抓完每本印出「選中章數」與「正文過短」警告。
