# novelAI — AI 輔助小說寫作系統

這個 repo 是「寫作系統套件」本身，不是某一本書；書資料夾（如 `gothic_witch/`）放在根目錄下（目前隨系統 repo 一起版控，見 README「開新書怎麼做」第 7 點）。完整說明見 `README.md`。

## 三個組成部分

- `結構定義/` — 各產物（摘要／世界觀／大綱／幕綱／章節／參照檔）的權威格式（schema），含所有 skill 共用的 `共同約定.md`。
- `技巧知識庫/` — 寫作方法（因果法則／結構公式／需求四象限／伏筆與意外／主題題旨／世界觀與背景／氛圍與筆調／選題與收斂／創作流程；`外部延伸/` 為非 Data 原文的補充，均標來源）。
- `.claude/skills/` — 操作：skill 承載「操作」、知識庫承載「方法」。

## Skill 地圖（作者在哪個階段，用哪個）

| 情境 | skill |
|------|-------|
| raw 很亂、主線/題旨/結局未定（粗階段） | `develop` |
| 大方向就緒，整理成大綱（全書或局部 arc） | `organize` → 審稿 `outline-test` |
| 大綱拆幕綱 | `beat-sheet` → 審稿 `beat-test` |
| 依幕綱寫正文 | `write` → 審稿 `write-test` → 修改 `revise` |
| 上游改動後找過期下游／回頭收斂前盤點已承諾事實 | `sync` |

## 沒有 skill 觸發時也要遵守

- 回答任何寫作技巧問題（對白、角色動機、結構、伏筆、POV……）之前，先查 `技巧知識庫/` 有無對應方法，依方法作答並註明出處檔名；知識庫沒有的，明說是 LLM 自身判斷。
- 修改任何書資料夾內的檔案之前，先讀 `結構定義/` 對應 schema 與 `共同約定.md`，遵守格式與「AI＝共創+審稿、非門檻」原則。
- 方法論忠實性：`技巧知識庫/` 的 Data 提煉檔不得混入自創說法；引用時以檔內文字為準（曾發生 skill 措辭與知識庫忠實性註記矛盾的漂移）。
