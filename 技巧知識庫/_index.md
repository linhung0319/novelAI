# 技巧知識庫索引

這份索引是知識庫的**檢索入口**。技法檔彼此是不同方法、扁平並列（不用資料夾分類——技法與 skill 是多對多，資料夾會強迫單一歸屬）；「層」欄只是輕量標籤，不是目錄。

**兩種用法：**
1. **臨場查找（discovery）**：沒有 skill 觸發、但要回答寫作技巧問題時（見 `CLAUDE.md`「回答任何寫作技巧問題前，先查技巧知識庫」），先讀本表的「何時查它」欄，定位到對的檔，再讀該檔對應小節。
2. **維護指針**：各 SKILL.md 用寫死的路徑＋章節錨點直達知識檔；改動技法檔時，對照「被哪些 skill 引用」欄回頭檢查指針。

## 「看情況二選一（OR）」的技法怎麼表示

有些技法不是普世規則，而是**看情況選一個**（結構公式、POV、文体、結局型、弧線型、開場型、以及網文↔文學的對衝軸）。本庫的一致處理機制是：

> **知識庫並列所有選項 → skill 把承重抉擇攤給作者拍板（不代決）→ 拍板結果寫進 `00-摘要.md`／`參照/` 的權威欄 → 下游 skill 讀該欄。**

**「選用決策」區塊慣例**：凡呈現 OR 選項的技法檔，用一致的欄位讓 skill 好取用——`選項 ｜ 這選項鎖定/適合什麼 ｜ trade-off（弱點）`，並在檔尾／檔首標明「拍板後記錄到哪個權威欄、哪個 skill 主持」。範本見 `結構公式.md`（並列＋選用建議）、`展示而非講述與五感描寫.md`（情境→建議表）、`網文與文學取捨.md`（對衝軸表）。

## 範例與分層政策（維護／擴充知識庫時遵守）

從 `Data/` 補萃取技巧、或改寫技法檔時遵守以下兩條，避免核心段重新膨脹、稀釋 skill 的檢索精準度。

**範例**
- **保留引用型範例**（引既有名作說明抽象方法，如《搶救雷恩大兵》《王者之聲》金庸）——它有 few-shot 幫助理解的好處，又不會讓小說產出僵化（講的是別的作品、非本書模板）。
- **每個原理留最強 1 個**：同一原理若堆疊 2–3 個效果相同的範例，精簡到一個；只有單一範例的小節不動。
- **禁止成品散文範本**：不放「要 LLM 照抄／模仿的整段輸出範文」。輸出格式屬 `結構定義/*.schema.md`，不進技法檔；一句話的教學對照片段（示範某招）可留。

**分層**
- **長檔核心／附錄分離**：判準地基（每次都要對照的方法）留核心；「這個 arc／這一幕不查也能寫」的深素材（跨語言補充、長版素材、附註、更多方法指路）下沉到「## 附錄（按需取用）」分隔線下。
- **日後新素材優先進附錄或只留指針**，不塞核心；>100 行的檔維護後同步更新檔頭「目錄」區塊，使其與實際 `##` 標題對齊。
- **為什麼**：`write` 一次載入約 15 個技法檔，核心愈薄、該幕真正需要的小節愈不會被淹沒（依 Anthropic skill 準則：reference 檔在被讀取前不佔 context，精準來自「只讀對的小節」而非「檔案小」）。
