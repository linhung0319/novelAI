# 技巧知識庫索引

這份索引是知識庫的**檢索入口**。技法檔彼此是不同方法、扁平並列（不用資料夾分類——技法與 skill 是多對多，資料夾會強迫單一歸屬）；「層」欄只是輕量標籤，不是目錄。

**兩種用法：**
1. **臨場查找（discovery）**：沒有 skill 觸發、但要回答寫作技巧問題時（見 `CLAUDE.md`「回答任何寫作技巧問題前，先查技巧知識庫」），先讀本表的「何時查它」欄，定位到對的檔，再讀該檔對應小節。
2. **維護指針**：各 SKILL.md 用寫死的路徑＋章節錨點直達知識檔；改動技法檔時，對照「被哪些 skill 引用」欄回頭檢查指針。

## 「看情況二選一（OR）」的技法怎麼表示

有些技法不是普世規則，而是**看情況選一個**（結構公式、POV、文体、結局型、弧線型、開場型、以及網文↔文學的對衝軸）。本庫的一致處理機制是：

> **知識庫並列所有選項 → skill 把承重抉擇攤給作者拍板（不代決）→ 拍板結果寫進 `00-摘要.md`／`參照/` 的權威欄 → 下游 skill 讀該欄。**

**「選用決策」區塊慣例**：凡呈現 OR 選項的技法檔，用一致的欄位讓 skill 好取用——`選項 ｜ 這選項鎖定/適合什麼 ｜ trade-off（弱點）`，並在檔尾／檔首標明「拍板後記錄到哪個權威欄、哪個 skill 主持」。範本見 `結構公式.md`（並列＋選用建議）、`展示而非講述與五感描寫.md`（情境→建議表）、`網文與文學取捨.md`（對衝軸表）。
