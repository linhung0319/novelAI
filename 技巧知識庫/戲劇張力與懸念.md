# 戲劇張力與懸念

> **用途**：本檔承載「方法」，收錄「讓讀者不停翻頁」的張力與懸念引擎：懸念型態、幕尾懸念結尾（cliffhanger）、悶燒蓄勢、以及讓讀者內心煎熬的四手法。供 `beat-sheet` 設計幕尾懸念、`write`／`write-test` 維持翻頁動力引用。
> **與其他檔的分工**（避免重複）：本檔管**張力／懸念／翻頁動力**；「意外／誤導／伏筆」的因果閉合歸 `伏筆與意外.md`；「前果＝後因」歸 `因果法則.md`；「兩難源於角色需求」的弧線設計歸 `需求四象限.md`；開場鉤子歸 `創作流程.md`／`場景與情節推進.md`。本檔只在交界處指路、不重述。
> **與網文的呼應**：幕尾懸念結尾＝網文的「절단（斷點/cliffhanger）」與「章末鉤子」的文學版依據，見 `網文與文學取捨.md`「連載鉤子」。

## 一、懸念＝閱讀的動力

「每一次閱讀都是出自一份好奇」（【戲劇手法】L67）。想讓讀者保持閱讀，就要先讓讀者保持好奇。懸念**不只是疑問**，有多種型態。

**兩種基本懸念型態**（【幕綱】L712）：
- **未知造成的好奇**：讀者不知道會發生什麼，想往下看求解。
- **已知造成的期待**：讀者已知某事將發生（好或壞），期待／擔憂它何時、如何發生。

> 這兩型可套用在開場（幕首勾起）與收尾（幕尾延續），核心要求一致——製造並維持好奇（【幕綱】L929）。

## 二、幕尾七種懸念結尾法（cliffhanger）

「幕首要勾起懸念，幕尾則要延續懸念」（【幕綱】L857）。在特定橋段**戛然而止**、不再敘述之後的反應，讓讀者往下翻。七種常用停止點（【幕綱】L851-869；【戲劇手法】L81）：

| 結尾法 | 停在哪 |
|---|---|
| 疑問 | 拋出問題、只給問題不給答案 |
| 高潮 | 停在情緒／衝突最高點 |
| 轉折 | 停在劇情反轉的當口 |
| 大事（做了壞事） | 停在重大事件剛發生 |
| 揭密（揭開秘密） | 停在秘密揭露的瞬間 |
| 決定（做了決定） | 停在角色做出關鍵決定 |
| 猶豫（正在猶豫） | 停在角色天人交戰、尚未決定 |

> 盡量讓每一幕收尾都安排一個停止點（【幕綱】L863）。這是 `beat-sheet` 設計幕綱「結果」欄與幕間銜接的直接依據；連載體（網文）尤其倚重此法維持追更。

## 三、讓讀者不停翻頁的六大手法（總覽）

洛克整理的六手法：**特點、目標、懸念、因果、悶燒、意外**（【戲劇手法】L14）。其中：
- **特點**：讓人物／題材有記憶點的獨特處。
- **目標**：角色有明確目標，讀者才有追看的方向（見 `因果法則.md`、`需求四象限.md`）。
- **懸念**：見本檔第一、二節。
- **因果**：事件有來由，讀者才不會「不知為何這樣」（詳見 `因果法則.md`）。
- **意外**：驚喜／驚奇（謊言、誤導、反轉）——詳見 `伏筆與意外.md`。
- **悶燒**：見下節。

## 四、悶燒（slow burn 蓄勢）

悶燒＝把張力壓住、緩緩加溫、延後爆發，讓讀者在等待中累積焦灼。是「延遲滿足」的張力技法，與 `網文與文學取捨.md`「節奏速度」對衝軸的**文學極**相通（對照網文的사이다快爽）。用得好能墊高最終爆點；用過頭則讀者流失——依「取向定位」拿捏悶燒的長度。

## 五、讓讀者心痛的四手法（內心煎熬）

「任何外部事件的衝擊都比不上內心煎熬」——內心煎熬是「小說的決勝點」，共鳴遠大於肉體疼痛（【戲劇手法】L347-350）。四手法：

1. **錯誤**：讓角色故意做錯的事、錯的選擇（如古惑仔越陷越深）。要訣：做壞事時**保留角色魅力**（為義氣／被逼迫／別無選擇），讀者才會同情而非厭惡；角色通常為錯誤付出代價（殘缺結局）。
2. **盲目**：角色對特定人事物堅信不移（明知不可為而為之／善良型盲目）。形成悲壯與浪漫，讓讀者揪心、被感動、顛覆其觀念邏輯。
3. **兩難**：精彩劇情的萬靈丹。分「有答案的兩難」（製造一陣緊張）與「沒答案的兩難」（折磨一輩子）。兩難的設計依據見 `需求四象限.md`「兩難的想要」（把兩難建立在角色真正在乎的東西上）。
4. **隱瞞（戲劇性反諷）**：「大家都知道，只有主角不知道」（如《楚門的世界》，連讀者都知道）。重點是**讓讀者知道悲劇即將發生**，恨不得衝進書中提醒主角。要訣在「**拖延**」——別讓悲劇太快發生或被解決，三番兩次快爆發又兜回懸而未決。

> 補充：一個負面結局（Bad End）也能最直接激起讀者心痛（【戲劇手法】L528）；但就算悲劇也該悲喜交加，讓讀者得到新的勇氣（【戲劇手法】L541）。

## 六、跨語言補充：開場與鉤子（幕首懸念的應用）

（以下為**跨語言策展筆記**提煉，非逐字原文；來源 `Data/跨語言技巧/開場與鉤子.md`。共同結論：**開頭不要倒設定、要製造好奇、要快**。）

**開場技法選單（英）**：強力第一句（用意外或懸念攔住，如 Orwell「clocks were striking thirteen」）／**in medias res（從動作正中間切入）**／令人一看的畫面／早定基調／早放引爆事件＋明確賭注／避免 info-dump。

**冒頭 5 型（日，依 genre 取捨的 OR 選單）**：①謎・疑問提示型（讓讀者立刻「なぜ？」，宜推理）②放り込む型（in medias res）③印象一文提示世界觀型（宜純文學）④語り手の声引き込む型（露出敘事者口吻）⑤日常中的違和感型。心法：一行目要有「引っかかるもの（勾住的東西）」、印象セリフ要一言、冒頭的リズム語感很重要。

**連載向開頭鉤子（網文，見 `網文與文學取捨.md`「連載鉤子」）**：簡中「黃金三章」＝聚焦主角、快速切入、製造期待感、早引金手指、設定融入情節、避免拖沓；韓文「第 1 話誘點第 2 話」＝第一段就給主角「當下的問題」、過去來由往後放、초반 후킹前 1–3 話要有強事件。

> 開場即「整本小說第一句話」的懸念設計——把第一、二節的懸念型態與結尾法反過來用在幕首（【幕綱】L838、L929「幕尾寫法移到幕首依然有效」）。開場的氛圍定調另見 `氛圍與筆調.md`「開場時間點決定氛圍」。

## 供哪些 skill 參照

- `beat-sheet`：設計每幕幕尾懸念結尾（七種停止點）、安排兩難與悶燒節奏；連載向的書依「取向定位」加強章末鉤子。「維持懸念、無一絲無聊」屬好幕七條件之一，是 `beat-test` 審幕綱時的隱性關注點。
- `write`：把幕尾懸念寫成實際的 cliffhanger；心痛四手法落到場景的內心煎熬描寫；開場鉤子寫成首章。
