# 結局類型

> **用途**：本檔承載「方法」，收錄結局的**六大要素**（每個結局都該追求的品質，AND）與**六大類型**（依效果二選一的選單，OR），供 `develop` 拍板「結局往哪走」承重分叉、`organize`／`beat-sheet` 排收尾、`outline-test`／`beat-test` 檢查結局與題旨一致引用。
> **與其他檔的關係**：結局類型的選擇**決定題旨**——見 `主題題旨.md`「結局決定題旨」；結局的「驚奇意外」要「意料之外、情理之中」——見 `伏筆與意外.md`；「大眾/爽 vs 文藝/深」的收尾取向見 `網文與文學取捨.md`。
> **元原則**：「好的結尾是故事成功的另一半……前面 90% 的故事內容都是為了結局而存在」（【結局】L175-178）。

## 一、好結局的六大要素（AND，越多越好）

除了讓讀者感受主題外，好結局應盡量達成這六項（【結局】L21-165）：

| 要素 | 說明 |
|---|---|
| 驚奇意外 | 結局別被輕易猜中；但須「意料之外、情理之中」，沿路埋合理伏筆——刻意超扯反而更慘（見 `伏筆與意外.md`） |
| 揭露人物 | 人物形象的意外（幫你的最後害你／害你的其實想幫你），比事件意外更震撼；但須先有跡象、非突兀 |
| 情感觸動 | 就算沒有意外，感情充沛一樣力挽狂瀾——「唯有真摯的情感才最動人」，餘韻繞梁 |
| 象徵意象 | 沒說出口的最感人；用具體事物承載情緒（可自創新意象，如《等一個人咖啡》烤香腸＝單戀）（完整象徵四類見本檔第四節；亦見 `展示而非講述與五感描寫.md`） |
| 圓滿呼應 | 首尾呼應：把故事中出現過、有特殊意義的物品／動作／地點／對白收回結尾（實用又安全，大力推薦每個故事都安插一個） |
| 觀念改變 | 結尾「再次安排一個選擇」，讓主角做出新選擇，把改變具體化（呼應角色弧線，見 `需求四象限.md`） |

## 二、六大結局類型（OR 選單，以「目標達成與否」界定）

選一種——依你想帶給讀者的感受決定（【結局】L187-331）。拍板後記錄於 `story/00-摘要.md`（結局走向影響「題旨」欄，見 `主題題旨.md`）。

| 類型 | 目標達成與否 | 帶給讀者／何時選 |
|---|---|---|
| **喜劇結局** | 達成目標 | 圓滿、正面力量；有情人終成眷屬、壞人死光 |
| **悲劇結局** | 沒達成目標 | 難過、無力、憤怒；角色雖敗但經歷努力挑戰，讀者掬同情之淚、有新體悟 |
| **悲喜劇結局** | 達成部分目標 | 笑淚交纏、五味雜陳；贏一部分輸一部分，比單純成敗更有深度、帶反思唏噓 |
| **暗示性結局** | 只暗示、不明說 | 讓故事有更多可能；有不同可能但「最恰當的」顯而易見（最恰當≠真相），重點在歷程而非目標 |
| **開放性結局** | 停在分歧、不說 | 「永遠折磨讀者」；在揭曉前愕然終結，留給讀者自行臆測 |
| **諷刺性結局** | 達成後才發現不是想要的 | 傳達價值觀反思；主角追表面目標（財富/成功），達成後回頭發現並不快樂（有的版本給回頭機會） |

> 一句話選用指南（【結局】L331）：想帶正面力量→喜劇；想讓人難過無力→悲劇；笑淚交纏→悲喜劇；想有更多可能→暗示；想永遠折磨讀者→開放；想傳達反思→諷刺。

## 三、兩個收尾特判：機器神與反高潮

- **機器神（Deus ex machina）**：靠突降的外力解決危機。現代視為不高明——破壞故事內的秩序與邏輯（【結局】L390），除非刻意用於喜劇/諷刺。與 `伏筆與意外.md`「意外必須源於主角努力、非機器神」同一條戒律。
- **反高潮**：刻意讓期待落空的收尾手法，有其專門訣竅（【結局】L478 起）；用得好可製造荒謬、諷刺或真實感，用不好則洩氣。需要時取原文細則。

## 四、餘韻四手法（讓讀者再三回味）

要讓作品被反覆詠讀，最根本仍是「故事中的情感」（`Data/【戲劇手法】.txt` L1108-1122「最核心的價值永遠都是故事中的情感」）；扣除劇情不談，另有四個讓「美」有跡可循的手法（`Data/【戲劇手法】.txt`「讓讀者再三回味的4大手法」L942-1105）。前兩者作用於**劇情/對白**，後兩者作用於**文句/意象**：

1. **隱晦**（有些事不說破比較好）：如一個有提示卻不公布解答的謎，讓讀者反覆咀嚼（「你到底有沒有愛過我？」「這是個好問題不是嗎」）。「說出來只是一個答案；不說出來則變成一個謎、一個折磨」。⚠️ 隱晦與「交代不清」只有一線之隔——大膽冒險、小心佈局。（劇情隱晦與 `伏筆與意外.md`「模糊／真假」相通。）
2. **留白**（自己想像比較過癮）：只給「觸發」與「結果」，把「過程」留給讀者想像（《三國演義》關羽溫酒斬華雄——出帳、喊殺聲、提人頭回帳、酒尚溫，過程全略）。想像力是小說最華麗的舞台。
3. **轉化**（硬是牛頭對馬嘴）：譬喻的雙胞胎兄弟——譬喻是「A 像 B」，轉化是「A 是 B」（擬物/擬人：「憂傷化成雨水從眼眶落下」）。用「就當作你是」的直接硬結合製造趣味。（譬喻見 `修辭與字句鍛鍊.md`。）
4. **象徵**（此物代表我的心）：以具體事物或形象間接表現抽象觀念——核心是「**有話別直說**」。四類：
   - **通俗象徵（普遍象徵）**：十人看到九人聯想相同（梅花／堅忍、烏鴉／霉運、白鴿／和平）；新穎但大眾能意會的也算（岳不群／偽君子）。
   - **特定象徵**：由作者在故事中創造，須看上下文才意會（朱自清《背影》＝父愛、《食神》愛心紙卡＝愛情開花結果、《等一個人咖啡》熱豆花＝相愛）。
   - **用行為象徵**：「洛克手捧九十九朵玫瑰等志玲下班」象徵愛意——同時避免小說「多說少演」。
   - **用物品象徵**：用小物件傳達意象（戴滿金戒指＝土豪、皮夾掉出保險套＝花花公子）。
   > 實戰入門：從「行為／物品」的意象開始練習。⚠️ 象徵的領悟權在讀者——「一切閱讀都是誤讀」（Umberto Eco）、「結局要由讀者決定」（李安），作者無法也不必控制所有解讀。

## 供哪些 skill 參照

- `develop`：結局是承重創作分叉（結局決定題旨），把六大類型當選項攤給作者拍板，回寫 `00-摘要.md`；並依 `網文與文學取捨.md` 定位判斷收尾偏大眾爽快還是文藝雋永。
- `organize`／`beat-sheet`：排收尾幕時對照六要素（尤其「圓滿呼應」預留首尾呼應點、「觀念改變」預留最後選擇）。
- `outline-test`／`beat-test`：檢查結局類型是否與摘要題旨一致、意外是否「情理之中」（有伏筆）、是否誤用機器神（只提可疑點、非門檻）。
