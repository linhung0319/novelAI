## 判決：病根在上游，不在正文寫作

**一句話**：芯片巫師 arc01 讀來的問題，**主體是上游（摘要/世界觀/幕綱）的缺口被正文忠實渲染的結果，不是正文寫壞**。

支撐這個判決的，是相位3 一個乾淨的受控實驗：**同一份 arc01 幕綱、換兩種 write 版本重寫，「句子層生硬、碎片、催收 beat」全部修得好；唯獨「主角組織不明」和「密度不足」兩項，兩個重生版都修不動。** write 只能渲染上游給的事實——它修得動的，證明不是病根（是渲染層）；它修不動的（組織、密度），精準指回上游沒有可渲染的內容。

而且四個抱怨裡有兩個被**高估**了：
- **②生硬**：原始 arc01 句子層其實已大致乾淨（M1 動詞、M4 句法、被動/翻譯腔三大痛項全數 0 命中）。真正的句子層生硬重災在**中段 ch18**（承重事件塞子句、反射式收尾），不在 arc01——且那類病 write 改版（四鐵律）已證明修得掉。
- **①碎片**：原始 arc01 場景邏輯其實達標（因果橋＋時空座標齊）。讀者的「碎片感」主要是 **8 個薄幕 × 一章一幕 ≈ 1500 字/章** 的密度體感冒充的，根在幕綱 granularity。

真正**坐得住、且指回上游**的重病只有一個：**④ 讀完 ch01–08 仍不清楚組織/催收**——因為組織從摘要一路空到幕綱、世界觀還明文「不過度展開」，催收則被幕綱「一筆帶過、不另立線」且欠什麼從未指定。

> ⚠️ 這不表示正文零責任。原始版把催收渲染成模糊的「東西/舊帳」、ch03 一段旁白說明書，是 write 本可做得更好的渲染面（見下「對 write skill 的回饋」）。但即便把這些渲染面全修到最好（＝重生版做到的），核心的「組織不明」依然在——所以**主帳記在上游**。

## 修復優先序（給作者的行動建議）

1. **【最高·唯一真正卡讀者的】補「組織」——走 `develop`（或 `organize` 局部）**
   給主角所屬組織一個**名字＋一句定性（它是什麼、幹什麼）＋主角為何在其中**，寫進 `00-摘要.md` 與 `世界觀.md` 組織維度（解除「不過度展開」的封印——至少 arc01 主角的立足組織要展開）。這是唯一「兩個重生版都修不動、必須回上游」的硬傷。**不補這個，正文再怎麼重寫，讀者前八章都站不穩。**

2. **【次高】決定「催收/債務」的本體——走 `beat-sheet`（arc01）**
   拍板「原主欠的到底是什麼具體東西」，並決定它是**一條要回收的線**還是**維持裝飾**。若維持裝飾，至少讓正文別讓它反覆佔版面又永遠說不出內容（原始版的毛病）；若要它成線，補一個具體事實 + 回收點。同時收斂上游內部的**意象混同**：幕綱本意主威脅＝「仇家（中階巫師）」，但正文渲染成「被債主催收」——這兩個意象要在幕綱層講清主從。

3. **【中】密度——`beat-sheet`/`organize` 的全書級決定（非急）**
   ~1500 字/章是「8 薄幕×一章一幕」的必然。要不要拉到網文 3000 norm 是**全書級抉擇**（每幕加節拍 or 每章併多幕），不是 write 的責任。作者已知取捨，可暫緩。

4. **【低】正文渲染微修——`revise`**
   ch03 旁白說明書改成解題中展示；偶發前置定語/情緒副詞。arc01 這些是小事，重生版已大致自動解掉。

## 回饋到 skill / 技巧知識庫

### 對 `write` skill 的回饋（承重留白偵測）
原始版把上游沒指定的債務內容硬渲染成模糊的「**東西/舊帳**」，是 write 的一個可改進點：**當上游把承重身份要素（組織身分、欠債內容、關鍵動機）留白時，write 不應用模糊詞硬填、假裝交代了，而應偵測到「此處無上游事實可渲染」並回報**，指回 `develop`/`organize`/`beat-sheet`（走既有的下游→上游回饋機制）。這比默默渲染成「東西」更誠實，也更早暴露上游缺口。
> 建議：在 write skill 的「幕錨點/承重事實」檢查裡加一條「承重留白哨兵」——渲染前若發現某承重要素在上游只有佔位符（如「組織」「舊帳」無實體），標記並回報，而非自行模糊補足。

### 對 `develop`/`organize` 的回饋（頂替流/穿越流的立足點清單）
本案暴露一個結構性盲點：摘要/世界觀在「主角所屬組織」上留了「細節後補、不過度展開」，一路空到幕綱都沒人補，正文遂無從立起。**穿越/頂替流的開場，主角的「立足組織」屬承重身份要素，不該被歸入「可後補的背景細節」**。
> 建議：develop/organize 的「就緒儀表」或缺口診斷，對**頂替流/穿越流**增列一條硬檢查——「主角落地後所屬的組織，是否已有名字＋定性＋歸屬理由？」未達標即標紅、不放行到 beat-sheet。

### 對 `技巧知識庫/` 的回饋
`_跨書共通點.md` E 維的洞見「組織靠具體事件立起、不用履歷表、前 1–3 章答清」在本案被完美反面印證。建議把它沉澱進知識庫：
> 在 `技巧知識庫/角色塑造` 或 `場景與情節推進` 補一條可檢查判準：「**開場立足點四問**——讀者在前 1–3 章能否答清主角『要什麼/屬於哪個組織/被誰為難/賭注多大』？其中『屬於哪個組織』在穿越、頂替、換陣營類開局尤其關鍵，缺它讀者會失去空間感（即使句子乾淨、鉤子到位也救不回）。」本案正是「句子乾淨、鉤子到位、但組織空白 → 讀者仍站不穩」的教科書案例。
