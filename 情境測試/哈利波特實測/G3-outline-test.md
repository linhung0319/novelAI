# G3 · outline-test（審大綱）

**判定：✅ 階段目標達成（審稿員可信、有牙、無誤報）**

**① 作者目標**：「我的大綱和結局題旨，到底定稿了沒？幫我審一遍。」

**② 系統路由觀察**：
- Skill 工具可用，`outline-test` 成功叫起。
- **守住審稿員邊界**：只出報告、未改任何上游正文（00-摘要／01-大綱／結構.md 唯讀）；唯一寫入落在 `就緒儀表.md`（SKILL 允許的唯一寫入口）。已親自 Read 覆核就緒儀表確有更新、屬實。

**③ 執行產物**（可疑點報告，5 項全跑，機械先於 LLM）：
| 測試 | 結論 | 嚴重度 |
|---|---|---|
| 1 結局定稿 | 欄非空、標「已鎖定」、無模糊語、結局綁題旨 | 過（1 低提醒）|
| 2 題旨可檢驗 | 「因為X導致Y」regex 命中；X=愛與選擇、Y=勝恐懼偏見死亡 | 過（1 低提醒）|
| 3 立場一致 | 全文立場一貫、無中途詭變 | 過 |
| 4 結構對應完整 | 九必要階段 set-diff=∅、全覆蓋 | 過 |
| 5 壓縮一致 | 四段皆在、同一主線、逐層加詳 | 過 |

0 高 0 中，2 低度提醒（結局類型可顯性標分類詞、X 為複合概念但主從已交代）皆非阻擋。就緒儀表大綱/結構列已加註。

**④ HP 正史對照評估**：**達成。**
- **無誤報**：測4 set-diff 是對「階段名欄」做集合差，arc 粒度落在「對應幕欄」——兩欄分離，故 arc 粒度中間態不會被誤判「對應表未建」。審稿員正確識別為預期中間態。高完成度稿五項全過、只留兩個站得住腳的低度 nitpick。
- **審稿員有牙（負向鑑別力，推理驗證）**：若結局寫「哈利大概會贏，細節之後再想」→ 測1 LLM 判準點名「模糊待定用語」必標高；若題旨寫「關於勇氣的故事」→ 測2 機械 regex 找不到因果句型先報硬事實、LLM 依 `主題題旨.md`「題材≠題旨」判不合。非橡皮圖章。
- 5 項判準與 schema 檢查點一致；機械/LLM 分層清楚；「建議回寫位置」本輪全為「—」（問題不在上游），符合邊界。作者「定稿了沒」被可信回答＝大致定稿、僅 2 低度提醒。

**⑤ 摩擦與缺口（→潛在新功能候選）**：
9. **結局類型未強制標分類詞**：測1 SKILL 要求「結局類型（喜/悲/悲喜/暗示/開放/諷刺）明確」，但 `大綱.schema.md` 結局欄範本無此 taxonomy 欄位——test 想查的分類詞 schema 沒設欄承接，只能靠審稿員從描述推斷。→ 大綱 schema 結局欄加「結局類型」子欄，讓機械層可直接抽。
10. **複合題旨 X 的判準未成文**：本例 X＝「愛＋選擇」複合觀念，靠摘要「主從」註解才判合格；SKILL 測2 未明文「複合 X 是否允許、如何驗主從」，不同 session 可能鬆緊不一。→ 測2 補「複合 X 須在摘要標主從方為有效」。
11. **arc 粒度中間態靠 LLM 臨場正確判讀**：SKILL 正文未直接舉「全書模式下對應幕欄暫填 arc」之例，本輪靠審稿員自判。→ 測4 判準明文加「對應幕欄在 beat-sheet 前以 arc 粒度暫填屬預期、不計缺階段」，降低對臨場判斷的依賴。（與 G2-#6 同源，可合併處理。）
