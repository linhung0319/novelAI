## 摩擦與缺口
- **arc02 是卷一唯一未 write-test 段**（S9 write 後收在 write→write-test 邊界·revise 留給下 session 但一直沒排上，直到 S15 補課）——垂直路線趕工時「write 完就往下一 arc」易漏掉回頭補審；write-test 覆蓋率靠台帳「下一步」逐條列選配才沒真漏。**候選觀察：管線是否需要一個「審稿覆蓋率盤點」提醒（哪些已成文段未跑 write-test）**。
- **新伏筆是 revise 階段才從正文兌現出來的**（beat-sheet/write 當時沒有）——上游幕綱、角色檔都要回頭補登＝下游成文反過來新增上游承諾，牽動 revise→sync→作者授權→幕綱套名一條鏈。分工鏈走順（revise 標待登記→sync 落角色檔＋指回作者→作者授權→sync 套幕綱），但**「下游兌現出新伏筆」是比「上游改動→下游過期」更難自動偵測的方向**（順向 sync 抓不到，靠 write-test/revise 人眼發現）。
- derived-sync CLI 是 Python（`uv run derived-sync`），非台帳舊記的 `node tools/derived_sync`——導演首次 spot-check 踩空一次（module not found），改 `uv run` 成功。台帳「`--book` 相對路徑坑」仍在（本次用絕對路徑）。
