## 摩擦與缺口

1. ⭐**新發現的污染類型（本 session 最重要的 honesty-guard 發現）**：`purity 無原著專屬細節`。**「原著」二字比 SNN 編號與 `fallback`／`[來源：腦]` 都嚴重**——SNN 洩漏的是編號、協定用語洩漏的是協定，**這一條洩漏的是測試的整個前提**（本書是照一本現成作品重建的）。已根除並列入常態掃描字表。教訓：**掃描字表本身需要定期擴充**，只掃已知的兩類會漏。
2. **SNN meta 洩漏根因已解、但根因未拔**：本 session 一次清光 82 處，但**產生它的機制沒變**——skill 在 `就緒儀表.md`／幕綱檔頭寫留痕時仍會沿用對話裡出現的編號。本輪對照組：兩個 worldbuild agent（S32）天然用「日期＋階段」，本輪 beat-sheet agent 也**全程未產生任何 SNN**（新建的 370 行 arc07 幕綱 0 處）——**目標寫法在無污染輸入下是自然發生的**。推論：污染是**傳播**而非**自發**，清乾淨後可望不再生。續盯。
3. **`beat-test` 的分級精度**：本輪唯一那條「高」被作者判為「中」且指出誤判（把「變形」讀成「蒸發」）。**但誤判仍有價值**——它挖出的真缺陷（引線沒被宣告成長線）作者自己也沒看見。這是第二次觀察到「**審稿的結論可能錯、指的方向卻對**」，值得記為 skill 的定位證據（審稿員非門檻，正是為此）。
