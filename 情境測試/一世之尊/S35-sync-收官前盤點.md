## 摩擦與缺口
- ⭐⭐**新發現·purity 級·git commit message meta 洩漏**：系統 agent 未讀 `examples/`／`情境測試/`（folder 防火牆守住），但盤點時**合法跑 `git log`**（sync 本就要查 git），而**書資料夾每次 commit 的訊息都以「端到端貫穿測試 SNN：…」開頭、meta commit 以「台帳：SNN…（commit hash …）」開頭**——`git log` 不分路徑顯示所有 commit 訊息，故系統 agent 由此**吸收了 S30/S34/「第十五輪 worldbuild」/commit 雜湊等測試 meta**（其報告直接引用「S30 / git f22082f」「第十五輪」即證）。**這是 S33「書資料夾 meta 根除 82 處」＋S34「file-scan 0 命中」基線未涵蓋的洩漏渠道**——12 詞 file-scan 只掃檔案內容·**不掃 git 歷史訊息**。嚴重度＝**中·辨識性/框架 meta（非原著 canon）**：核心防火牆（examples/情境測試/canon）未破·未洩任何劇情正解·對本 session（機械 sync）產出品質無影響；但系統 agent 因此「知道自己在一個多 session 測試裡」＝§9「辨識性洩漏」實例。**修法選項（方法論級·留作者拍板·本 session 未擅改 commit 慣例以維持 git 歷史一致）**：(a) 書資料夾 commit 訊息改為純 in-world（不含 SNN/端到端字樣）·SNN→hash 對映改由 `session台帳` 承載（台帳＝meta·系統讀不到）＝根治但破 S1–S34 訊息一致性；(b) 測試 meta commit 移到獨立 git 歷史/另一 repo；(c) 接受 git-log 為已知洩漏渠道·在系統 agent prompt 明列「git log 內的 SNN/端到端字樣是外部標記·忽略之」（弱）。**傾向 (a)**，但屬慣例變更、應由作者（測試主）定奪。
