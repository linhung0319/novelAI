## 導演方法論（三角色防火牆執行）
- **系統 agent 冷啟／續用**：本 session 用了 **2 個系統 agent 實例**：
  - 寫作 agent（WR-R1 提案＋WR-R2 落筆＋加厚＋就緒儀表收尾）＝**同一實例續用**。判準沿用 S6：WR-R1→WR-R2→加厚屬同一「動筆」步驟被 elicitation 切成數段，該 agent 全程未觸 examples/情境測試、載完整 context，續用避免重建風險；就緒儀表更新亦屬同一 write 收尾。
  - **write-test agent＝全新獨立冷啟**（審稿必須另起，保客觀、不為自寫產物護航）。✅
- **作者 agent 冷啟／續用**：用 **1 個作者 agent 實例**（Q1–Q6 首答＋字數密度待決點續答）＝續用同一實例。判準：兩輪同屬 arc01 write 的同一場 elicitation，且第二輪「知會點3 真慧水下鉤觸發素材」直接關聯作者首輪剛拍的 Q5，續用保決策一致；作者 agent 讀 examples 本就不破防火牆，續用無有效性風險。
- **逐字轉述**：系統↔作者間全程逐字轉問轉答、導演未注入原著。Q1–Q6、字數待決點皆原文轉。
- **導演 spot-check**：導演讀 ch0001／ch0006（書資料夾內、不破防火牆）核實 write-test 留痕與 purity——確認 ch0006 幕009 高度 POV 越界屬實、半形標點屬實、ch0001 腔調基準紮實且無原著專屬洩漏。

## 摩擦與缺口（真作者會想要、系統接不住或值得留意）
1. **字數基準需作者一次拍板才對齊**：系統首寫全 arc 一致偏短 ~25–30%（皆完整節奏單位、非骨架化），是「精實模式」而非張力驅動的重分配；作者拍 (b) 局部加厚後才對上網文向密度。→ **未來 write 可在動筆前先問「本 arc 目標字數/密度檔位」**，省一輪回修。
2. **半形標點機械缺陷**：加厚章（ch0001/02/04）為 0，但 ch0005/06（第一輪原寫、未加厚）殘留 267 處半形標點——疑首輪落筆的輸入法/生成殘留。→ **候選新功能：write/write-test 落檔前自動跑全形標點正規化（純機械、無創作判斷）**，避免污染正文地板。
3. **高度 POV 越界屬「渲染層」而非結構**：write-test 判可由作者直改 ch0006 幕009 文字（不必動幕綱/beat-test）。→ 顯示 POV 知識邊界在正文層比幕綱層更難守（貼身第一人稱＋想寫出氛圍張力時，容易替讀者「定性」不可知之物），呼應台帳「write 前跨 arc 對齊：外景級威能留白」的同類守則——**感知留白同理，是正文層反覆風險點**。
