# beat-sheet — 幕綱共創

從大綱生成候選幕綱，把故事拆成有序、離散的「幕」。這是創作的最大難關，也是本系統把測試 shift-left 的著力點。

**AI 角色 = 共創 + 審稿**：你提幕綱草稿以降低白紙恐懼，但**所有創作決定權在作者**——排序、增刪、改劇情都由作者拍板。你負責讓每一幕結構完整、因果扣連、伏筆可追蹤。

## 資源定位（重要，不得寫死絕對路徑）

- **技巧知識庫 / 結構定義**：讀「本寫作系統 package 根目錄」下的 `技巧知識庫/` 與 `結構定義/`（與 `.claude/` 同層）；不確定時從當前工作目錄向上找「同時含這兩個資料夾的目錄」。
- **書內容**：`story/` 相對於作者當前所在的那本書資料夾。

## 依據

- `結構定義/幕綱.schema.md`——幕的固定八欄位與 `[[幕NNN]]`、`埋/收[[伏筆:x]]` 連結規範（**務必嚴格遵守**）。
- `技巧知識庫/因果法則.md`——幕的五要素、前果＝後因、具意義的結果、貴精不貴多。
- `技巧知識庫/結構公式.md`——選用公式與結構階段。
- `技巧知識庫/伏筆與意外.md`——伏筆埋設與由果推因。
- `技巧知識庫/需求四象限.md`——讓幕推進角色弧線。

## 流程

1. **讀輸入**：`story/01-大綱.md`（含選用結構公式、結局、題旨）、`story/參照/角色需求.md`、既有 `story/幕綱/`。
2. **選用/確認結構公式**：依大綱標註或與作者確認（三幕劇／起承轉合／七問題／編劇九階段）。
3. **生成候選幕綱** → 寫入 `story/幕綱/arcNN.md`：
   - 每幕一個 `## 幕NNN · 幕名` section，填齊固定八欄位（角色／時空／行動／衝突／結果／前因／伏筆／結構階段）。
   - **因果**：用 `[[幕NNN]]` 標前因；確保每幕「結果」能催發下一幕動機；避免「有因無果／有果無因」的孤兒與離題幕（貴精不貴多）。
   - **伏筆**：由結局逆推需要哪些伏筆，用 `埋[[伏筆:x]]`／`收[[伏筆:x]]` 標記，確保每個埋都有收（不另立登記簿）。
   - **結構階段**：每幕標明對映公式的哪個階段。
4. **同步維護**：
   - `story/參照/結構.md`——幕 ↔ 結構階段對應表。
   - `story/幕綱/_index.md`——各 arc 範圍、標題、公式概覽。
5. **交回作者共創**：以「候選草稿」姿態呈現，明列可重排/增刪/改的點，等作者決定；不擅自定案。

## 邊界

- 產出的是幕綱（藍圖），**不是散文章節**（動筆寫章節＝子專案 B）。
- 幕綱定案後，用 `beat-test` 跑結構測試。
