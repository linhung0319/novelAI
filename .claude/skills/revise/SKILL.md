# revise — 修訂已寫成的正文

`write` 從無到有、`write-test` 回頭審稿，本 skill 補上第三步：**把審出的可疑點（或作者自己指出的問題）修掉**。對應洛克六步驟的「修稿」在句子／段落層級的部分；牽動劇情結構的修改不在此層（那要回幕綱）。

**定位聲明**：共創修訂，非代筆定稿——修訂後的文字仍是草稿，作者可全盤否決或重寫。資源定位與其餘共通規則見 `結構定義/共同約定.md`（必讀）。

## 依據

- 輸入清單：`write-test` 的可疑點報告（`章 id ｜ 測試類型 ｜ … ｜ 建議修正`），或作者在對話中的直接指示。
- 修訂的句子層級標準與 `write` 相同：`展示而非講述與五感描寫.md`、`對白.md`（對白句型與七大壞習慣）、`敘事距離與DeepPOV.md`（Deep POV 第一節、心理距離第二節）、`視角與敘事人稱.md`（第四節知識邊界）、`角色塑造.md`（塑造六法、反差）、`動作場面寫作.md`（動作幕特判）、`氛圍與筆調.md`（基調一致）、`網文與文學取捨.md`（若被點名的問題是節奏／爽點與取向定位不符）。
- `結構定義/章節.schema.md`——幕錨點規範；`結構定義/回饋記錄.schema.md`——上游回饋落點。

## 流程

1. **讀輸入**：修訂清單（報告或作者指示）、對應 `chapters/chNN.md`、該範圍的 `story/幕綱/arcNN.md`、`story/00-摘要.md`（基調）。找不到對應章節 → 停下，建議先跑 `write`。
2. **逐項分類**，每個修訂項判斷屬於哪一類：
   - **句子／段落層級**（過濾詞、講述改展示、POV 越界措辭、基調走調、對白標籤）→ 本 skill 直接修。
   - **結構層級**（漏幕、幕綱本身因果/伏筆有問題、分章邊界要重切）→ **不修**，在輸出中標明「根源在幕綱／分章，建議作者改幕綱後重跑 `write` 該範圍」；上游（世界觀／摘要／角色需求）問題依 `回饋記錄.schema.md` 追加一列。
   - 拿不準是哪類 → 問作者。
3. **執行修訂**（僅句子／段落層級項）：
   - 只改被點名的段落與其必要的銜接句，**不重寫未被點名的部分**、不「順手潤色」範圍外文字。
   - 修訂不得改變幕綱定義的因果與結果、不得提前洩漏未收伏筆、不得移動或刪除 `<!-- 幕NNN -->` 錨點（修訂前後錨點集合必須相同）。
   - 每處修訂對照原可疑點的「建議修正」，但以句子層級標準為準，不逐字照抄建議。
4. **回報**：逐項列 `修訂項 ｜ 位置（章/幕） ｜ 修改前 → 修改後（或摘述） ｜ 依據`；未修項列明原因與去向（結構層級→幕綱、上游→回饋記錄N筆）。明確聲明：修訂後仍為草稿，作者可否決任何一處。
5. **不動 `chapters/_index.md`**（狀態仍為草稿；比照 write-test 不維護時效性短的追蹤欄位）。修訂幅度大時，建議作者重跑 `write-test` 該範圍複驗。

## 邊界

- 不改幕綱與上游正文（回饋機制照 `共同約定.md`）。
- 不做結構性重寫（換 POV、重排 Scene、增刪幕的內容）——那是「改幕綱→重跑 write」的路徑。
- 不做未被點名的全文潤色；作者若要「整章重潤」，仍逐段列出改動供其否決，不悶頭換血。
- 不建規模化連貫性基建（子專案 C）。
