# 書本模板

這是「一本新書」的資料夾模板。**複製整個 `書本模板/` 到 repo 根目錄，改名為你的書名**（例如 `奇幻冒險/`），就能開始一本新書。

> 版控現況：`.gitignore` 的 `/*` 規則目前被註解，書資料夾會直接進系統 repo 版控（要獨立版控見系統 README「開新書怎麼做」第 7 點）。

## 用法

1. 複製本資料夾 → 改成書名（放在 `novelAI/` 根目錄下）。
2. 把你的瑣碎筆記、靈感、半成品（任意格式）丟進 `raw/`。
3. 對 Claude 說：**「幫我把 raw/ 整理成符合結構的大綱和幕綱」** → 觸發 `organize`，產出/更新 `story/` 並回報缺口診斷。
4. 反覆用 `beat-sheet` 共創幕綱、用 `beat-test` 審稿，直到幕綱通過測試。
5. 幕綱就緒後，對 Claude 說「幫我把這幾幕寫成正文」→ 觸發 `write`（子專案 B）：AI 先提議分章方案供你拍板，再依幕綱展開成 `chapters/` 下的散文草稿。寫完用 `write-test` 審稿、`revise` 修訂。
6. 回頭改了上游之後，說「哪些下游過期了」→ 觸發 `sync` 列出待重驗清單。
7. 讓小說內容有版本歷史：目前 `.gitignore` 的 `/*` 規則被註解，書資料夾會直接進系統 repo 版控；要獨立版控見系統 `README.md`「開新書怎麼做」第 7 點。

## 結構

```
你的書/
├── raw/        # 你丟進來的任意格式素材（靈感、筆記、半成品）
├── story/      # 整理後符合結構的產出
│   ├── 00-摘要.md
│   ├── 01-大綱.md
│   ├── 幕綱/
│   │   ├── _index.md
│   │   └── arc01.md
│   └── 參照/
│       ├── 角色需求.md
│       └── 結構.md
└── chapters/   # 依幕綱寫的章節草稿（子專案 B，由 write 產生）
    ├── ch01.md
    └── _index.md
```

`story/` 與 `chapters/` 下各檔的權威格式定義見系統套件的 `結構定義/*.schema.md`（含 `章節.schema.md`）。本模板的檔案只是空骨架，實際內容由 `organize` / `beat-sheet` / `write` 依 schema 填寫。
