# 書本模板

這是「一本新書」的資料夾模板。**複製整個 `書本模板/` 到 repo 根目錄，改名為你的書名**（例如 `奇幻冒險/`），就能開始一本新書。

> 書資料夾與系統套件一起在本 repo 版控（刻意設計，見系統 README「開新書怎麼做」第 7 點）——開新書後記得 commit。

## 源／衍生兩種檔（新設計，先懂這個）

本系統的設定與摘要採**源→衍生兩檔**（見 `結構定義/共同約定.md` 二）：

- **源檔 `<實體>.md`**（人管）＝你自由手寫的唯一真實來源，**無格式要求**、好讀好改。設定層一實體一檔：`story/設定/角色/<名>.md`、`story/設定/世界觀/<主題>.md`、`story/設定/風格/風格.md`。
- **衍生檔 `<實體>.ai.md`**（AI 管）＝skill 依源檔重生的結構化分析（如 `.c`→`.o`），**你不手改**；改設定去改源檔、再讓 skill 重生。它只會「新鮮／過期」，不會與源漂移。
- **你只碰源檔**（`.md`，非 `.ai.md`）＋作者利器（`01-大綱.md`、`幕綱/`）＋章節正文（`chapters/chNNNN.md`）。

## 用法

1. 複製本資料夾 → 改成書名（放在 `novelAI/` 根目錄下）。
2. 把你的瑣碎筆記、靈感、半成品（任意格式）丟進 `raw/`。
3. **看 raw/ 收斂到哪，選一句說**（新書多半是前者）：
   - raw 還很亂、主線／題旨／結局未定 → **「幫我把 raw/ 想清楚」** → 觸發 `develop`，陪你把大方向談清楚，產出源 `00-摘要.md`＋`設定/世界觀/<主題>.md`、`設定/角色/<名>.md`、`設定/風格/風格.md`，並重生對應 `.ai.md`。**這是新書的正常起點**——`develop` 刻意不產大綱與幕綱。
   - 大方向已大致就緒 → **「幫我把 raw/ 整理成符合結構的大綱」** → 觸發 `organize`，產出 `01-大綱.md` 並回報缺口診斷。
   - 只有某一段想清楚、全書結局還沒定 → 「先幫我把『這一段』整理成大綱」→ `organize` 局部模式，產出 `大綱/arcNN.md`，不必替未定的結局搶先拍板。
4. 大綱／摘要大致定稿後可用 `outline-test` 審；再反覆用 `beat-sheet` 共創幕綱、用 `beat-test` 審稿，直到幕綱通過測試。
5. 幕綱就緒後，對 Claude 說「幫我把這幾幕寫成正文」→ 觸發 `write`（子專案 B）：AI 先提議分章方案供你拍板，再依幕綱展開成 `chapters/` 下的散文草稿（正文 `chNNNN.md`＋衍生 metadata `chNNNN.ai.md`）。寫完用 `write-test` 審稿、`revise` 修訂。
6. 回頭改了上游之後：機械偵測跑 `cd tools/derived_sync && uv run derived-sync check --book <你的書>`（列出哪些 `.ai.md` 過期）；語意過期（結局→伏筆、世界觀→幕可行性）說「哪些下游過期了」→ 觸發 `sync`。
7. 小說內容的版本歷史：本資料夾隨系統 repo 一起版控，開新書、寫到段落時記得 commit（衍生 `.ai.md` 也一起 commit）。

## 結構

```
你的書/
├── raw/        # 你丟進來的任意格式素材（靈感、筆記、半成品）
├── story/      # 整理後符合結構的產出
│   ├── 00-摘要.md          # 源（人管）：主線／題旨／結局／基調／視角／取向的承重決定
│   ├── 00-摘要.ai.md       # 衍生（AI）：50/100/200 字壓縮／高概念／取向定位分析
│   ├── 01-大綱.md          # 作者利器·直改
│   ├── 設定/               # ★ 設定層：一實體一檔（源＋衍生）
│   │   ├── 角色/
│   │   │   ├── <角色名>.md      # 源：自由手寫這個角色
│   │   │   ├── <角色名>.ai.md   # 衍生：需求四象限／弧線／馬斯洛／🧊
│   │   │   └── _index.ai.md     # 衍生 rollup：全書角色清單
│   │   ├── 世界觀/
│   │   │   ├── <主題>.md        # 源：一主題一檔（魔法／組織／時間…；仙俠即功法招式）
│   │   │   ├── <主題>.ai.md     # 衍生：限制與代價／影響力／自洽／升格哨兵
│   │   │   └── _總覽.ai.md      # 衍生 rollup：七維度盤點／核心規則索引／待確認矛盾
│   │   └── 風格/
│   │       ├── 風格.md          # 源：這本書該用什麼腔調寫
│   │       └── 風格.ai.md       # 衍生：禁用詞表／稱謂系統／可用句式＋腔調分析
│   ├── 幕綱/               # 作者利器·直改
│   │   ├── _index.md
│   │   └── arc01.md
│   └── 參照/               # 綜合衍生檔（AI 寫、不走 hash）
│       ├── 就緒儀表.md
│       └── 結構.md
└── chapters/   # 依幕綱寫的章節草稿（子專案 B，由 write 產生）
    ├── ch0001.md       # 源：純散文正文＋內嵌幕錨點 <!-- 幕NNN -->
    ├── ch0001.ai.md    # 衍生：對應幕／POV／基調・風格參照／狀態
    └── _index.ai.md    # 衍生 rollup：權威章序
```

`story/` 與 `chapters/` 下各檔的權威格式定義見系統套件的 `結構定義/*.schema.md`（含 `角色.schema.md`／`世界觀.schema.md`／`風格.schema.md`／`章節.schema.md`）。

> ⚠️ **上面這些檔在模板裡都是空骨架，不代表你已經有大綱或幕綱。** 它們只是先把位置佔好、讓你看得出這本書最後長什麼樣；每個檔的檔頭都標了「尚未產出」，由 `develop` / `organize` / `beat-sheet` / `write` 依 schema 逐步填寫。看到 `01-大綱.md` 存在就以為可以直接跳去拆幕，是這個模板最容易誤導人的地方。
>
> **設定層 `設定/角色/`、`設定/世界觀/` 一開始是空的**（只有 rollup 佔位）——要加角色/設定，就在對應資料夾新增一個 `<名>.md`／`<主題>.md`，愛怎麼寫怎麼寫，skill 會據此生成 `.ai.md`。
>
> **`.ai.md` 檔尚未由 skill 產出前**，`derived-sync check` 會把它們報成 `unstamped`——這是「還沒生成」的正常狀態，不是錯誤。
