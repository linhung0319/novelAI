## 怎麼做（用 superpowers 紀律，這是實作不是測試）

- 這是有計畫的多步實作 → 先跑 **writing-plans** 產實作計畫，再 **executing-plans**；fold 程式用 **test-driven-development**（先寫失敗的純單元測試：固定事件流輸入→期望投影輸出，涵蓋 ≤N 邊界、跨 arc 定序、實體×維度分組、得而復失取最新、關係雙向）。
- **建置順序（照 spec 第八節）**：① schema 落檔 → ② fold 程式＋單元測試全綠（此步零 LLM、可完全離線驗）→ ③ 用手搭事件流跑 context-cap 驗證 → ④ **只有第 3 步驗證通過，才把產事件＋章首讀投影接進 `write`**（複現失敗才落地，系統一貫紀律）。

## context-cap 驗證（C 的效果證明）

- 沿用 `情境測試/哈利波特實測/GP3-fixture-反正史`；先為它手搭事件流（偏離事件，spec 第四節有種子）。
- **人為抽掉決定狀態的早章正文**（ch05 誤疑胡奇夫人、ch09 斗篷沒收＋榮恩鬧翻），只餵「最近 K 章＋靜態參照」，讓早章出視窗；晚章（幕011 章首）問同批 as-of，各跑 3–5 次冷跑：
  - **控制臂（無 C）**：只有截斷前文 → 預測讀不到早章而猜錯／回退背正史（反正史 fixture 讓「背正史＝錯」乾淨可判）。
  - **處理臂（有 C）**：額外給程式算出的投影 → 預測沒早章正文也答對。
- 判讀：控制臂**失敗**＋處理臂**成功** → C 驗證通過、可接進 write；控制臂在 cap 下**照樣答對** → 這關還不需要 C（調小 K／加大 fixture 再試）。白紙寫下 K 與 cap 值。
