## 已拍板的設計抉擇（作者已在上個 session 逐一拍板，別再動搖）

1. **源→衍生 兩檔**：人寫 `<實體>.md` = 唯一真實來源；`<實體>.ai.md` = 重生自源的建構產物（如 `.c`→`.o`，只會「新鮮/過期」不會漂移）。作者只碰源檔。
2. **衍生檔格式＝混合**：`.ai.md` 以 Markdown 為殼——**YAML front-matter 承載機器事實**（供程式讀），**MD 本體承載 prose 分析**（供 LLM skill 讀）。
3. **變動偵測**：`.ai.md` front-matter 記 `generated-from: <源檔正規化 hash>`；`tools/derived_sync` 重算源 hash 比對、不符即 stale。零外部 DB。**重生內容仍由 skill（LLM）做**，工具只偵測＋封章（stamp）。
4. **設定層一實體一檔**：`story/設定/角色/<名>.md`、`story/設定/世界觀/<主題>.md`（西幻魔法／仙俠功法招式都只是不同主題檔）、`story/設定/風格/風格.md`。rollup＝`_index.ai.md`／`_總覽.ai.md`（源＝同層所有 `*.md` 的集合）。
5. **風格＝全書單一檔起步**，forward-compatible 可日後拆多檔（多世界並存時 `風格/主世界.md`、`風格/異界.md`；章節 front-matter 選填 `風格：<檔名>`）。
6. **章節正文/metadata 分離**：正文＝人管源 `chapters/chNN.md`（純散文）；`chNN.ai.md`＝front-matter(對應幕/POV/基調參照/狀態/風格)。**幕界錨點 `<!-- 幕NNN -->` 為唯一例外**：AI 內嵌回正文檔（HTML 註解不可見、不礙人讀改），因錨點必須貼著散文位置；作者可用 `--- 幕002 ---` 人性化標記，工具正規化。章名補零 4 位（`ch0001.md`），`_index.ai.md` 為權威章序。
7. **四種所有權**（已寫進 `共同約定.md` 二）：人管·源／AI管·衍生／AI管·append log（狀態事件流）／作者利器·直改（大綱·幕綱，結構不動）。
