# 結構定義：結構（`story/參照/結構.md`）

> **這是什麼**：記錄本書選用哪一套結構公式，以及每個必要階段對應到哪幾幕，作為 `beat-test`「結構完整性」測試的基準。
> **方法依據**：`技巧知識庫/結構公式.md`。
> **所有權**：**AI管·衍生（宣告式，非 hash 追蹤）**。它是「選用公式＋幕↔階段對應」的綜合視圖、由 `beat-sheet` 從幕綱的「結構階段」欄同步維護、無單一源檔可 hash 對映，故**不使用 `.ai.md` 命名、不被 `tools/derived_sync` 掃描**（與 `就緒儀表.md`／`狀態事件流.md` 同屬「AI 寫、作者不手改、但不走 hash 偵測」的綜合檔；見 `共同約定.md` 二所有權表，列 `結構` 時不帶 `.ai.md`）。幕綱改動後本表是否過期，走 `sync` 語意比對＋`beat-test` 對照，非 hash。

---

## 用途

把「選用的結構公式」與「幕 ↔ 結構階段的對應」寫成一份權威表，讓 `beat-test` 能對照檢查：缺哪個必要階段、階段重複、或錯序。此檔由 `beat-sheet` 在拆幕時同步維護。

## 檔案格式

```markdown
# 結構

## 選用公式
編劇九階段（見 技巧知識庫/結構公式.md）

## 各必要階段 ↔ 幕對應
| 階段 | 對應幕 | 備註 |
|------|--------|------|
| 平凡失衡 | 幕001–幕003 | |
| 召喚衝突 | 幕004–幕005 | |
| 推動冒險 | 幕006–幕008 | |
| 覺悟改變 | 幕009 | 跨越不可逆門檻 |
| 磨練成長 | 幕010–幕023 | 佔比最多 |
| 信心重挫 | 幕024 | 最終陰影 |
| 低潮醒悟 | 幕025–幕026 | |
| 瀕死重生 | 幕027 | 最後之戰 |
| 昇華圓滿 | 幕028 | |

## α（作者自訂的變形／轉折點）
（選填：三大轉折點、N/V 型情緒曲線等自訂加料。）
```

## 規則

- **選用公式**須是 `結構公式.md` 收錄的其中一套（三幕劇／起承轉合／七問題公式／編劇九階段／Save the Cat! 15 節拍表／雪花法），或作者自訂但註明依據。
- 對應表要覆蓋該公式的**每一個必要階段**；缺階段是 `beat-test` 的重點警訊。
- 記住元原則「結構是死的，人是活的」——允許 α 變形，但需在此標明，避免被誤判為錯序。

## 檢查點（供 beat-test 參照）

- 選用公式的必要階段是否都有幕對應？（缺 → 結構不完整）
- 是否有階段重複或明顯錯序（與公式順序牴觸且無 α 說明）？
- 幕綱各幕的「結構階段」欄位是否與本表一致？
- **節奏檔位相稱性**（讀 `00-摘要.md`「取向定位」的節奏檔位子維度，本檔不另存）：各 arc/結構階段所標的檔位（開頭段／常態段）是否與其結構位置相稱——早期階段（平凡失衡／召喚衝突）多為開頭段（高密度事件驅動），中段（磨練成長）多為常態段（靠長線大謎供能、章末鉤降級）？宣告錯位（中段仍套開篇緊繃、或開頭就走常態鬆弛）是 `beat-test`／`outline-test` 的警訊。詳見 `技巧知識庫/戲劇張力與懸念.md` §六(d)。
