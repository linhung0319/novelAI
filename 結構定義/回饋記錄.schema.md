# 結構定義：待裁決回饋（各衍生 `.ai.md` 檔末的共用「## 待裁決回饋」區）

> **這是什麼**：「動尾改頭」在本系統的具體落地格式。當 `outline-test`／`beat-test`／`beat-sheet`／`write`／`write-test`／`expand` 等下游流程在細節層發現「問題根源其實在上游設定／摘要」，先落地記錄於此，交作者裁決是否回寫**源檔**，**不自動改動源、也不改衍生檔的分析區**。
> **本次重設計的核心變更**：回饋不再附加在**源檔**末尾（源檔是人管、好讀好改、無 schema）。改為附加在對應**衍生 `.ai.md`** 的「## 待裁決回饋」區——因為衍生檔本就會被下游 skill 讀作 context，且它會在重生時被覆蓋，回饋隨「消化→改源→重生」自然清空。定調見 `共同約定.md` 四。
> **方法依據**：本檔屬**系統延伸設計，非 `Data/` 原文方法**。精神來自 `技巧知識庫/創作流程.md`「回到哪一層修／動尾改頭」與全系統「AI 是審稿員不是門檻」的哲學。

---

## 用途

創作是遞迴的：作者可能已經在拆幕、甚至已經在寫，才發現世界觀某條規則不夠用、或摘要的題旨其實沒站穩、或某角色的「需要」中段就滿足了。若沒有地方接住這種「下游倒逼上游」的發現，只能靠作者自己記得要回頭改——本區塊就是那個接住的地方。

## 哪些衍生檔帶這個區塊

帶「## 待裁決回饋」區的是**有上游源檔可回寫**的衍生檔：

| 衍生檔 | 對應源檔（回寫標的） | 誰回頭消化 |
|--------|--------------------|-----------|
| `story/設定/世界觀/<主題>.ai.md`、`_總覽.ai.md` | `世界觀/<主題>.md` | `worldbuild` |
| `story/設定/角色/<名>.ai.md`、`_index.ai.md` | `角色/<名>.md` | `character` |
| `story/設定/風格/風格.ai.md` | `風格/風格.md` | `develop` |
| `story/00-摘要.ai.md` | `00-摘要.md` | `organize`／`develop` |
| `chapters/chNNNN.ai.md`（罕見） | 章節正文 `chNNNN.md` | `write`／`revise` |

- **幕綱沒有回饋區**——它是「作者可直接修改的利器」（`作者利器·直改`）；發現幕綱問題只在對話中指出，建議作者直接改或重跑 `beat-test`，不代寫。
- **`結構.md`／`就緒儀表.md`／`狀態事件流.md`** 是綜合衍生檔、無單一上游源可回寫，不帶本區塊。

## 格式（各衍生 `.ai.md` 檔末統一附加）

```markdown
## 待裁決回饋
| 日期 | 來源 | 發現 | 建議回寫（改哪個源檔） | 狀態 |
|------|------|------|----------------------|------|
```

- **日期**：發現當天。
- **來源**：哪個 skill／哪次測試（例：「beat-test 測試4（幕017–020）」「beat-sheet 排 arc03 時」「outline-test 測試1」「write-test 測試9（ch0007）」——承重細節促升；「write（ch0007 撰寫時·承重細節缺口）」——撰寫時撞到設定缺口、暫定發明的承重細節待裁決是否入設定）。
- **發現**：一句話講清楚問題是什麼。
- **建議回寫（改哪個源檔）**：具體指向要改**哪個源檔**的哪一段／哪一條（如「角色/凱.md：其『需要』中段就被滿足，弧線塌陷」），不需要寫好完整替代文字。
- **狀態**：`待裁決` ／ `已採納`（附一句改了源檔哪裡）／ `已駁回`（附一句原因，避免下次重複回報）。
- **角色檔可加「對象角色」欄**：`角色/<名>.ai.md` 是一角一檔、對象已由檔名確定，通常無需加欄；`_index.ai.md` 若記跨角色問題可加「對象角色」欄標明是哪個角色。

## 規則

- **只記錄，不代寫**：下游 skill 發現問題可以**直接追加一列**到對應衍生檔的本區塊，這是唯一允許下游 skill 直接寫入別人衍生檔的動作；**源檔**（規則、四象限、題旨、腔調……）的實質修改仍須作者確認後才進行，不由 skill 自動代勞。
- **改的是源、不是衍生分析區**：回饋提請的是「改源檔」；源改了才重生衍生檔，回饋列隨重生清空（或轉「已採納」註記後於下次重生清）。**絕不**繞過源、直接手改衍生檔的分析區——那會在下次重生時被覆蓋、且違反源→衍生單一真實來源。
- **消化前先看回饋**：`worldbuild`／`character`／`develop`／`organize` 精修某源檔前，先看對應衍生檔末本區塊有無待裁決項，一併消化再動筆，避免回饋石沉大海。
- **不另立登記簿**：回饋就地附加在被影響的衍生檔末尾，不另開一份跨檔案的回饋總表（呼應 `幕綱.schema.md` 的「單一真實來源」精神）。
- **就緒儀表的感知規則**：任一 skill 在其正常流程更新 `story/參照/就緒儀表.md` 時，若順手發現對應衍生檔有「待裁決」列，於「產物軸 × 深度成熟度」表的備註欄加註「⚠️N 筆待裁決回饋（見 X.ai.md）」。這是被動偵測，不需要為此另開一輪全檔掃描流程。
