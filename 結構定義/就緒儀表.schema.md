# 結構定義：就緒儀表（`story/參照/就緒儀表.md`）

> **這是什麼**：追蹤各「產物軸」的成熟度與就緒清單，讓作者看得出「哪裡夠成熟可以往細節下沉、哪裡還粗」。由 `develop`／`organize`／`outline-test`／`beat-sheet`／`beat-test` 更新，下沉與否由作者拍板。
> **方法依據**：本檔屬**系統延伸設計，非 `Data/` 原文方法**。精神來自 `選題與收斂.md` 的「概念→雛型→選拔」成熟分層、與 `創作流程.md`「回到哪一層修／動尾改頭」。
> ⚠️ **非門檻**：本系統一貫「AI 是審稿員不是門檻」。就緒清單是**定性診斷**，不是分數或自動 pass/fail。

---

## 用途

創作不是把「粗」100% 想完才進「細」。就緒儀表把成熟度攤成**兩軸**，讓局部下沉成為可能：

- **產物軸**（半獨立成熟、互相餵養）：世界觀／主線+題旨+基調／角色（需要+弧線）／結構（公式+階段）。
- **深度軸**（由粗而細）：粗（大方向）→ 大綱 → 幕綱。

同一本書，世界觀可能已鎖定、主線卻還草擬——儀表就是要讓這種**不平均**被看見，避免用單一門檻卡錯地方。

## 檔案格式

```markdown
# 就緒儀表

## 產物軸 × 深度成熟度
| 產物軸 | 粗（大方向） | 大綱 | 幕綱 | 備註 |
|--------|------------|------|------|------|
| 世界觀 |  | — | — | |
| 主線＋題旨＋基調 |  |  |  | |
| 角色（需要／弧線） |  |  |  | |
| 結構（公式／階段） | — |  |  | |

（每格填：空白／草擬／就緒／已鎖定。「—」＝該軸不適用該深度。）

## 就緒清單（下沉「大綱」前參考，定性、非門檻）
- [ ] 能用一句話說清主線
- [ ] 題旨可寫成「因為 X 導致 Y」
- [ ] 結局方向已拍板
- [ ] 主角／視角已定，且主角「需要」明確
- [ ] 世界觀無擋路的硬矛盾（見一致性提示）
- [ ] 基調已定（一句話）

## 局部下沉紀錄（以細促粗）
- 局部下沉可以發生在**兩個深度層**，各自留痕，也可疊加：
  - **大綱層局部下沉**：某 arc 的大方向已收斂，但全書結局／題旨未定 → `organize`（局部模式）產出 `story/大綱/arcNN.md`（不是 `01-大綱.md`），見 `大綱.schema.md`「局部下沉（scoped）大綱」節。
  - **幕綱層局部下沉**：某 arc 已有大綱依據（`story/01-大綱.md` 或對應的 `story/大綱/arcNN.md`）→ `beat-sheet` 拆成 `story/幕綱/arcNN.md`，其餘 arc 仍留在較粗的層。
  - 常見路徑：同一 arc 先在大綱層下沉、確認後再於幕綱層下沉；也可能兩層一次做完。
- 下沉品一律標「暫定（待粗層鎖定）」，並記錄：依據哪些既有材料下沉、範圍界線在哪、明確排除哪些部分。
- 例：「開場序列（arc01）已具體到可下沉：大綱層 → `story/大綱/arc01.md`；幕綱層 → `story/幕綱/arc01.md`（幕001–007）。其餘 arc 仍留粗階段。」
```

## 規則

- **不設數值門檻**：清單全勾不是「通過」，只是「風險較低」；作者可在未全勾時仍決定下沉。
- **成熟度是局部的**：允許對已清楚的小段先下沉做細，其餘留粗。下沉成品須標「暫定（待粗層鎖定）」。
- **大綱欄可記錄至 arc 粒度**：全書大綱未定稿時，若已有一或多個 arc 完成大綱層局部下沉，大綱欄可填「草擬（僅 arcNN／…）」而非留白，寫法比照幕綱欄。
- **動尾改頭**：粗層一旦更動，回頭重驗已下沉的細段（呼應 `創作流程.md` 的分層回修）。
- 由 `develop`（粗）、`organize`／`outline-test`（大綱）、`beat-sheet`／`beat-test`（幕綱）各自在其階段更新對應格與清單。
- **回饋感知規則**：任一 skill 在其正常流程更新本檔時，若順手發現 `世界觀.md`／`00-摘要.md`／`角色需求.md` 有「下游回饋記錄」待裁決列（見 `回饋記錄.schema.md`），於對應列的備註欄加註「⚠️N 筆待裁決回饋（見 X.md）」。這是被動偵測，不需另開全檔掃描流程。
