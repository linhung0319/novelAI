# 結構定義：章節（`chapters/chNN.md`）

> **這是什麼**：定義「幕綱寫成散文後」長怎樣。這是子專案 B（撰寫）的核心 schema——把幕綱的結構化欄位（角色／時空／行動／衝突／結果）展開成有場景、有對白、有感官細節的正文段落，同時保留可回溯到幕綱的錨點，讓因果與伏筆的機械可追溯性不因進入散文層而中斷。
> **方法依據**：`技巧知識庫/場景與情節推進.md`（Scene/Sequel 展開）、`展示而非講述與五感描寫.md`（句子層級執行）、`視角與敘事人稱.md`（POV／知識邊界）、`敘事距離與DeepPOV.md`（Deep POV／心理距離調控）、`對白.md`（對白句型與壞習慣）、`動作場面寫作.md`（動作場面專項，選用）、`畫面感與電影化.md`（運鏡／示現／插敘／轉場）、`修辭與字句鍛鍊.md`（譬喻／擬詩／映襯／字句，依取向定位選用）、`心理描寫.md`（內在描寫五技法＋疼痛描寫）、`網文與文學取捨.md`（取向定位決定節奏、爽點密度與文字功底門檻）。

---

## 用途

章節＝依作者拍板的分章方案，把連續幾幕的幕綱內容展開成正文。**章節是幕綱的散文呈現，不是取代品**——幕綱仍是唯一的結構真實來源（因果鏈、伏筆狀態）；章節正文只是它的「渲染結果」，可隨時重寫，不影響幕綱本身的正確性。

## 檔案配置

- 每章一個檔：`chapters/ch01.md`、`ch02.md`……**全書連續編號，不因跨 arc 而重置**。
- 索引檔 `chapters/_index.md`（見下）。
- **編號穩定**：已寫成的章節原則上不重新編號；分章方案調整只發生在尚未動筆的範圍。

### 章號不預配號段（與幕號的不對稱是刻意的）

`幕綱.schema.md` 替每個 arc 預配 100 個幕號，**是因為幕號是連結標的**——`[[幕NNN]]` 前因、`埋/收[[伏筆:x]]` 所在的幕、正文錨點 `<!-- 幕NNN -->` 全都指著它，重編號得追著散落各處的引用改，代價極高、明令禁止。

**章號不是連結標的**：沒有任何產物用 `[[chNN]]` 指向章節，章號只活在檔名與 `chapters/_index.md`。所以章號**不預配號段**，`write` 也**不替尚未寫成的上游 arc 猜測預留章號**——猜測本身才是風險（猜少了會與已寫的章相撞，猜多了留下一段永久空洞）。

亂序下沉、先寫後段的 arc 時：從當下最小的可用章號往後編，`_index.md` 以「（未寫）」列標出前面 arc 的幕範圍。日後補寫排在前面的 arc，若作者要讓章號回到閱讀順序，改**檔名、檔頭標題、`_index.md`** 三處即可，沒有引用要追。挪不挪由作者決定——`_index.md` 的「對應幕」欄才是章與幕的權威對映，章號只是檔案的名字。

## 單一章節的格式

```markdown
# ch01 · 禁忌的叮嚀與焚夜

- 對應幕：[[幕001]]–[[幕004]]
- 所屬 arc：arc01
- POV：小女巫（第三人稱有限視角／Deep POV）
- 基調參照：見 `00-摘要.md`「基調」— 哥特恐怖＋黑色幽默＋黑暗童趣
- 狀態：草稿

---

<!-- 幕001 -->
（幕001「禁忌的叮嚀」的 Scene 正文……）

<!-- 幕001→幕002 -->
（銜接幕001到幕002的 Sequel／過渡正文，若份量足以成獨立段落才另起錨點……）

<!-- 幕002 -->
（幕002「溜進畫廊」的 Scene 正文……）

……
```

### 欄位說明

| 欄位 | 說明 |
|------|------|
| 對應幕 | 用 `[[幕NNN]]`–`[[幕MMM]]` 標示本章涵蓋的連續幕區間；一個章節理論上對應一段**連續**的幕範圍，不交錯覆蓋其他章節已涵蓋的幕 |
| 所屬 arc | 對照 `story/幕綱/arcNN.md` |
| POV | 本章固定的敘事視角角色＋人稱/視角類型；v1 每章僅一位 POV 角色（不同章節可切換 POV，同章內不可） |
| 基調參照 | 不重抄基調全文，只引用 `00-摘要.md`，供撰寫與未來 write-test 核對一致性 |
| 狀態 | 草稿／已定稿（`write` 產出一律先標「草稿」；作者確認後可手動改標） |

## 幕錨點規範（機械可抽取，呼應幕綱 `[[幕NNN]]` 的精神）

- **`<!-- 幕NNN -->`**：標記緊接其後段落主要在渲染該幕的 Scene（Goal/Conflict/Disaster）正文。同一幕若跨多段，只需在第一段前標一次，直到下一個錨點出現前都視為同一幕的延伸。
- **`<!-- 幕NNN→幕MMM -->`**：標記兩幕之間的 Sequel／過渡正文（銜接前一幕結果到下一幕動機的反應/兩難/決定）。若過渡文字很短（一兩句話），可省略獨立錨點、併入前一幕錨點段落尾端。
- **不新增第二套 ID 系統**：一律沿用幕綱既有的 `幕NNN` 三位數編號——正文層的追溯基準仍是幕綱編號，呼應幕綱「單一真實來源」的精神。
- **一幕不可跨章節切開**：章節邊界只能落在某個 `幕NNN` 錨點之前或某個過渡錨點之前，不可把同一幕的 Scene 正文切成兩半分屬兩章。

## 章密度規範（分章單位＝一個節奏單位）

- **章＝一個節奏單位**，有兩種、都合法：(i) 一個**閉合**的爽點/情緒循環（小起承轉結），或 (ii) **一段連續張力線中的一節**——本章只結一個小節拍，那條宏觀目標**跨章不閉合、持續悶燒**。分章單位是「節奏單位」，不是「篇幅感」。**不是每章都得閉合**——整段（數章）都各自閉合小勝利、章末各自打包，正文就會碎、無記憶點（見下「反模式②」與 `技巧知識庫/場景與情節推進.md` §六）。
- **依 `00-摘要.md`「取向定位」定基準**：**中文網文向預設一章一幕的粒度**（一幕＝一個 Scene）、**約 3000 字**（起點體例）＋章末鉤子；文學向才容許章節自然收束、跨幕邊界較鬆。**「一章一幕」只定粒度、不等於「每章都要閉合一個完整爽點循環」**——該幕的 Scene 可只推進一條跨章張力線而不了結它。
- **兩個反模式（皆避免）**：①把兩個各自完整的爽點循環壓進一章 → 蒙太奇→承重事件被塞進子句→下一章引用即「跳躍、要回頭查」（字數遠超 3000 是信號）；②每章各自閉合小勝利、**章末總結式收尾**、宏觀張力從不跨章 → 碎片化、無記憶點。
- **章末三種切法（`write` 撰寫章邊界時選一種；細則見 `技巧知識庫/戲劇張力與懸念.md` §二）**：**硬切斷章**（下一章緊接同一張力→停在未解畫面/動作、不回顧不預告）、**前開式軟鉤**（用具體新擾動向前拋）、**收束斷章**（段/arc 邊界才閉合＋喘息、鉤子仍是新擾動非回顧）。**反模式「總結式收尾」明令避免**：章末用「回顧本章處境＋預告未來」的抽象總結段收尾（例：「他站在一個死人的屋子裡…欠著死人的債」／「三日之後，真正的那個人要親自來了」）＝講述，是「硬要總結感」的來源；續接章根本不該收攏、硬切即可。
- **字數依張力起伏重分配、非等長勻平**（3000 字為基準非死線，依段落自然邊界浮動）：一段的**高潮幕鋪成 set-piece**（厚）、**過場幕壓薄**（快），並安排喘息/日常節拍與峰後餘韻讓記憶點長出來（見 `場景與情節推進.md` §六）。某幕遠不足基準且與鄰幕合成一節奏單位→可併章（作者拍板）；某幕遠超基準含多循環→訊號是幕綱該拆幕（回 `beat-sheet`），受「一幕不可跨章切開」限制、`write` 不硬拆。
- 方法依據見 `技巧知識庫/網文與文學取捨.md`（節奏速度·連載鉤子強度）、`場景與情節推進.md` §五（單章快慢）§六（跨章張力線·快慢交替·餘韻）、`戲劇張力與懸念.md` §二（章末三種切法）、`Data/跨語言技巧/結構與節奏.md`（連載單位）。
- 上方「單一章節的格式」示例（ch01·幕001–幕004 四幕一章）為**文學向較鬆邊界之例**；中文網文向預設一章一幕的粒度。

```markdown
| 章 | 對應幕 | 所屬 arc | POV | 狀態 | 備註 |
|----|--------|----------|-----|------|------|
| ch01 | 幕001–幕004 | arc01 | 小女巫 | 草稿 | |
| ch02 | 幕005–幕007 | arc01 | 小女巫 | 草稿 | |
| （未寫） | 幕008–… | arc02 | — | — | arc02 幕綱尚未拆出/尚未撰寫 |
```

- 每次 `write` 新增/更新章節時同步維護本表。
- 表尾可加一列標示同一 arc／全書尚未寫成章節的幕範圍，避免作者誤以為已全部完稿。

## 章末狀態快照（`chapters/_index.md` 選用區塊）

> **⚠️ 定位更新（子專案 C v1）**：本區塊自子專案 C 起**降級為衍生產物**——狀態的單一真實來源改為 `story/參照/狀態事件流.md`（見 `結構定義/狀態事件流.schema.md`），章末快照是「fold 投影程式（`tools/state_projection`）從事件流算出的 as-of 投影」，**非權威、非手抄、可重算可丟棄**。原「由 write 手維護」的做法在 `write` 接上事件流後退場（見該 skill）。保留本區塊格式定義，供投影輸出與人眼閱讀對照。

逐章記錄 POV／關鍵角色在「**該章結束時**」的狀態，由 `write` 寫完每章順手維護（見 `write` skill 收尾步驟）。直接服務三件事：**Deep POV 資訊控制**（此刻 POV/讀者知道什麼、仍不知什麼）、**伏筆連貫**（哪章誰碰過/知道某物）、**關係演變刻度**。

- **這是一條「別處沒有」的獨立軸，不是第二本帳**：`角色需求.md` 記的是**靜態弧線**（每角一份、非逐章時間線）、幕綱「本 arc 伏筆狀態」表記的是**埋/收機制**、`sync` 反向記的是**已承諾事實**——都不追蹤「章末此刻誰知道什麼/持有什麼」。本區塊只補這個角色視角切片；**引用伏筆用名字即可（如「713 包裹」），不重抄幕綱伏筆表的埋/收狀態**（避免副本漂移，呼應「單一真實來源」）。
- **定位邊界（重要，勿僭越子專案 C）**：只做**單集／連續章內的輕量快照**，不是跨 arc/跨集的狀態機、不做一致性自動校驗——那些是子專案 C 的規模化基建。本區塊是它近乎零成本的前身，**已於子專案 C v1 併入**（真實來源移至 `story/參照/狀態事件流.md`，本快照為其可重算投影）。
- **只記已寫章**；未寫章不預填。最承重的是「知識前沿」一格，其餘按需填。
- 格式（每個已寫章一則，接在索引表後）：
  ```markdown
  ## 章末狀態快照
  ### ch02（章末）
  - 知識前沿：小女巫已知自己是被老婦人收養、村裡大人刻意迴避「收養」這件事；**尚不知**自己的身世真相、也不知老婦人隱瞞的原因
  - 關係刻度：與老婦人（表面順從、暗生疏離）；同齡玩伴尚未登場
  - 持有／狀態：帶著養母給的舊布包；**尚未**察覺自己的能力
  ```
  （範例用中性泛例，非任何書內容——schema 範例一律不引某本書的實際章節，以免污染以該書做盲測的驗證。）

## 規則

- **正文修改權在作者**：`write` 產出的是可直接編輯的草稿，不是受保護的定稿；沒有「本體 vs 回饋記錄」的分權設計（不同於摘要/世界觀/角色需求）——因為章節沒有比它更下游的產物需要回饋於此。
- **沒有下游回饋記錄區塊**：本檔案不附加 `結構定義/回饋記錄.schema.md` 的區塊。撰寫時若發現問題根源在幕綱本身，走「建議作者直接編輯幕綱或重跑 beat-test」；若溯源到世界觀/00-摘要/角色需求，走這三份檔案既有的下游回饋記錄機制，不在章節這層另開一份。
- **不做自動一致性追蹤**：幕綱若在章節寫成後又被修改，本 schema 不要求自動比對／自動標記過時（規模化連貫性追蹤留給子專案 C）；`write` 至多在對話中提醒作者留意，不建立比對基礎設施。

## 檢查點（供未來 write-test 參照）

- **幕涵蓋完整性**：本章「對應幕」範圍內每一幕，是否都能在正文中找到至少一個 `<!-- 幕NNN -->` 錨點？
- **POV 一致性**：本章是否全程維持單一 POV 角色、無 head-hopping？POV 角色是否敘述了自己不該知道的事（知識邊界洩漏）？
- **展示而非講述＋對白**：關鍵時刻（衝突高點、結果轉折、情緒反應）是否以展示而非講述呈現，過濾詞是否已清除？對白有無七大壞習慣、每句是否幹活（見 `對白.md`）？
- **動作場面專項**：若本章有幕屬於戰鬥/動作類型，是否套用動作場面寫作的專項準則；是否誤將「詞彙鋪張」「鋪墊框架」套用到非玄幻/仙俠類型的一般動作幕？
- **基調一致性**：本章筆調是否與 `00-摘要.md` 基調一致，無不自覺轉調？
- **節奏與定位一致**：本章實際節奏／揭露速度／資訊密度／鉤子強度，是否與 `00-摘要.md`「取向定位」欄宣告的網文↔文學傾向相符（見 `網文與文學取捨.md`）；文字是否達「流暢四層次」地板（無錯別字、直白、順口、無時代錯亂詞彙破壞氛圍——第②③層的**操作化句法細則**見 `技巧知識庫/字詞句法與抗生硬.md`，並由下一條「句法生硬／翻譯腔」把關）？
- **句法生硬／翻譯腔**（取向定位無關的普世地板）：句子本身是否生硬——結構可能全對，但字詞句法仍讀來澀、有翻譯腔／作文腔？對照 `技巧知識庫/字詞句法與抗生硬.md` M7 十病徵（動詞名詞化／抽象名詞裸奔／的的不休／翻譯腔嵌套／被動濫用／代詞冗餘／前置副詞貼標籤／形容詞描述聲音／喻體疏離／語域錯配；命中率最高＝M4 句法、M1 動詞）。與「展示而非講述」正交（那管該不該演，本條管句子澀不澀）。詳見 `write-test` 測試10。
- **畫面感與內在描寫**（選用，視段落性質）：是否出現「停滯鏡頭」（單一事物過度描述拖慢節奏）、示現/插敘是否中斷了讀者當下最想看的高潮（見 `畫面感與電影化.md`）？心理描寫是否「掛在鮮活人物身上」、還是像醫學說明書／無來由的意識流（見 `心理描寫.md`）？修辭（譬喻/映襯）是否老套、喻依是否貼合氣氛（見 `修辭與字句鍛鍊.md`；文學向段落才重點檢查）？
- **分章合理性**：本章起訖點是否落在自然的場景/時間/結構階段邊界，而非任意切斷一個 Scene？
- **章密度/節奏**：本章是否＝一個節奏單位（閉合循環，或連續張力線之一節；依取向定位；中文網文向預設一章一幕的粒度、約 3000 字＋章末鉤子）？有無把兩個循環擠進一章（蒙太奇·反模式①）？字數是否**依張力起伏重分配**（高潮 set-piece 厚、過場薄），還是每幕等長勻平？有無喘息/日常與峰後餘韻（記憶點）？
- **章末切法/跨章張力線**：章末是否為**總結式收尾**反模式（「回顧本章處境＋預告未來」的抽象總結段）？續接同一張力的章是否**硬切**（未解畫面/動作、不回顧不預告），還是硬收攏＋總結？整段（數章）是否**每章各自閉合小勝利、宏觀張力從不跨章**（碎片化·反模式②），還是有一條連續張力線橫拉、持續悶燒（見 `write` skill 五鐵律 R5、`場景與情節推進.md` §六、`戲劇張力與懸念.md` §二）？
- **連續性與場景/摘要比**：承重事件（幕的行動/衝突/結果）是否展開成讓讀者經歷的 Scene，還是被摘要進子句/意義敘述（讀者沒經歷到）？相鄰段落/相鄰章之間有無時空/因果斷裂？有無反射式收尾標記段段宣布意義（見 `write` skill「正文五鐵律」R1/R3/R4/R5）？
- **賭注兌現／角色反應可信度（渲染層）**：幕綱設計好的重大賭注（威脅／秘密曝光／底線被踩）在正文渲染時有沒有被「降溫偏誤」寫軟——反派無故退縮／主角無動機放行／挑起的威脅在後文蒸發？（賭注是否兌現的**設計層**把關在 `beat-test` 測試6；本檢查點是**渲染層**——見 `技巧知識庫/因果法則.md` §七。）
