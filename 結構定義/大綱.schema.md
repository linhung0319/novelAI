# 結構定義：大綱（`story/01-大綱.md` 或 `story/大綱/arcNN.md`）

> **這是什麼**：定義「符合結構的大綱」長怎樣，作為 `organize` 產出與 `beat-sheet` 拆幕的輸入。本檔定義**兩種變體**：預設的全書大綱 `story/01-大綱.md`，與局部下沉（scoped）大綱 `story/大綱/arcNN.md`（呼應 `就緒儀表.schema.md` 的局部下沉原則、比照 `幕綱.schema.md` 的每-arc 一檔慣例）。兩者擇一使用，視全書結局/題旨是否已收斂而定；規則見文末「局部下沉（scoped）大綱」章節。
> **方法依據**：`技巧知識庫/創作流程.md`（大綱＝完整故事、兩次發想兩次收斂）、`技巧知識庫/結構公式.md`、`技巧知識庫/主題題旨.md`、`結構定義/就緒儀表.schema.md`（局部下沉原則）。

---

## 用途

大綱是**把摘要擴寫成結構完整、真正的故事**：連續敘述、不講究文筆修辭，直觀地把情節一一寫下。這是創作的最大難關（創造故事的「結構」）。**結局與主題在此階段必須已定。**（本節與其後「檔案格式」「規則」「檢查點」皆指**全書版** `story/01-大綱.md`；**局部下沉版** `story/大綱/arcNN.md` 見文末專節，定稿門檻不同。）

## 檔案格式

```markdown
# 大綱

## 選用結構公式
（開頭標註：三幕劇／起承轉合／七問題公式／編劇九階段之一，見 結構公式.md。）

## 故事全文（連續敘述）
（從頭到尾把故事講一遍。情節依因果排列，結局與主題明確。
 給自己看的草稿，不必雕文筆；建議別超過三千字。）

## 結局與題旨
- 結局：……
- 題旨：因為（X），導致（Y）。
（結局是回頭修正前文的地方——改結局就要回頭檢視前文邏輯。）
```

## 規則

- **結局與主題已定**：沒有結局，大綱不算定稿（`organize` 缺口診斷應標「還沒有結局 → 大綱無法定稿」）。
- **開頭標註選用的結構公式**，供後續 `beat-sheet` 對映幕、`beat-test` 做結構完整性測試。
- **連續敘述**：這是一個精簡的故事，不是幕的列表（拆幕是幕綱的事）。
- 兩種大綱版本（見 `創作流程.md`）：本檔是**創作用大綱**（無長度限制、可含草綱→細綱的演進）；對外宣傳大綱不放這裡。
- **結局是回頭修正前文的樞紐**：改結局 → 回頭檢視前文因果與伏筆（呼應「動尾就要改頭」）。

## 檢查點（供 outline-test 參照）

- 結局是否明確？題旨是否可寫成「因為 X 導致 Y」？
- 故事的立場是否自始至終一致（無中途詭變）？
- 選用的結構公式是否標明，且大綱大致依該骨架推進？

## 局部下沉（scoped）大綱：`story/大綱/arcNN.md`

> 對應 `就緒儀表.schema.md` 的核心原則：「成熟度是局部的：允許對已清楚的小段先下沉做細，其餘留粗」。當**全書**結局/題旨還沒收斂，但**某個 arc**已清楚到可以往細節走，用這個變體，不必為了走流程而被迫替尚未收斂的部分搶先拍板。

### 何時用這個變體，而非 `01-大綱.md`

- 全書結局／題旨／主線**尚未**全部拍板（`01-大綱.md` 無法定稿），但**特定一段（arc）**已具備連續、因果明確的材料，足以支撐拆幕。
- 一旦全書結局/題旨拍板、`01-大綱.md` 可以定稿，**不再新增**新的 scoped 大綱片段（既有片段的收斂規則見下）。

### 檔案配置

- 每個已下沉的 arc 一個檔：`story/大綱/arc01.md`、`arc02.md`……**arc 編號與 `story/幕綱/arcNN.md` 共用同一識別碼**——同一段故事從大綱層到幕綱層，arc 編號不變。
- 索引檔 `story/大綱/_index.md`。

### 檔案格式

```markdown
# arc01 · ＜arc 名稱＞

> ⚠️ **暫定，待粗層鎖定**。
> 範圍說明：本 arc 對應哪些已收斂的材料，以及**明確排除**哪些仍在粗階段、本 arc 不處理的部分。

## 選用結構公式（本 arc 適用）
（同全書大綱；若全書公式尚未拍板，標「暫定，僅套用本 arc」。）

## 本段全文（連續敘述）
（只寫這個 arc 範圍內的情節，依因果排列。範圍是「這一段」，不是整本書。）

## 本段收束與鉤子
- 本段收束：（這個 arc 自己的因果收尾——是「這一段」的階段性結果，**不是**整本書的結局）
- 鉤子／懸而未決：（本段刻意留給後續 arc 或全書結局解決的伏筆／問題）
- 對全書結局／題旨的關係：未定 ／ 暫定方向為……（誠實標注，不得讓本段收束偽裝成全書結局）
```

### 與全書版的差異

| | 全書版 `01-大綱.md` | 局部版 `大綱/arcNN.md` |
|---|---|---|
| 範圍 | 整本書 | 單一 arc |
| 定稿門檻 | **結局與題旨已定**（硬性） | **本段收束已定**即可；全書結局可仍「未定」 |
| 「結局」欄位 | 全書結局 | 改用「本段收束與鉤子」，須誠實標注對全書結局的關係 |
| 狀態標示 | 定稿後即為正式大綱 | 恆標「⚠️ 暫定，待粗層鎖定」，直到併入 `01-大綱.md` 或粗層鎖定 |

### 與全書 `01-大綱.md` 的收斂規則

- 全書結局/題旨拍板、`organize` 要產出/更新 `story/01-大綱.md` 時，**必須**先讀取既有 `story/大綱/arcNN.md`（如同讀 `raw/`），把內容併入連續敘述。
- 併入後，原檔**不刪除**：檔頭加註「⚠️ 已併入 `01-大綱.md`（日期），本檔轉為歷史存檔，正文以 `01-大綱.md` 為準」，保留沿革記錄。
- `story/大綱/_index.md` 同步該 arc 列註記「已併入」。
- 併入後，`beat-sheet` 一律改讀 `story/01-大綱.md`。

### 索引檔 `story/大綱/_index.md`

```markdown
# 大綱索引（局部下沉）

- arc01：＜arc 名稱＞ —— 狀態：暫定，待粗層鎖定
- arc02：＜arc 名稱＞ —— 狀態：已併入 01-大綱.md（2026-08-01）
```

### 檢查點（scoped 版，供 outline-test 參照）

- 「本段收束」是否明確？
- 「鉤子／懸而未決」是否誠實列出，未偽裝成已解決？
- 選用結構公式是否標明（可暫定）？
- 「對全書結局的關係」是否誠實標注未定／暫定？
