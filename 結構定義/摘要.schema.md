# 結構定義：摘要（源 `story/00-摘要.md` ＋衍生 `story/00-摘要.ai.md`）

> **這是什麼**：定義「符合結構的摘要」長怎樣，作為 `organize`／`develop` 產出與 `outline-test`／`beat-test` 檢查的權威格式。
> **本次重設計的核心變更**：摘要改成**源→衍生兩檔**——源 `00-摘要.md` 承載作者拍板的**承重決定**（主線一句話／題旨／結局方向／基調／視角結構／取向定位），自由手寫、無格式要求；衍生 `00-摘要.ai.md` 承載 AI 從源重生的**壓縮與分析**（50/100/200 字壓縮、高概念 Look/Hook/Book、取向定位逐軸分析），front-matter 放機讀事實供下游快取。所有權見 `共同約定.md` 二。
> **方法依據**：`技巧知識庫/創作流程.md`（摘要＝主線的壓縮測試）、`技巧知識庫/主題題旨.md`（題旨公式）、`技巧知識庫/選題與收斂.md`（概念→延伸→聚焦）、`技巧知識庫/氛圍與筆調.md`（基調）、`技巧知識庫/網文與文學取捨.md`（取向定位）、`技巧知識庫/視角與敘事人稱.md`（視角結構）。

---

## 用途

摘要是**主線**（故事劇情的大致走向），不是主題。它是一項**壓縮測試**：寫不出一句話，代表你還沒真正搞懂自己的主線。承重的創作決定（主線、題旨、結局、基調、視角、取向）在源檔由作者拍板；壓縮與分析交給 AI 從源重生。

## 源檔 `00-摘要.md`（人管，無 schema，承載承重決定）

作者自由手寫即可——**沒有格式要求，重點是好讀好改**。以下是「該想清楚並寫下的承重決定」（是內容清單、不是欄位格式；愛用什麼排版都行）：

```markdown
# 摘要

## 一句話（主線）
（一句話講清這是什麼故事。寫不出來＝主線未定，回頭想。這是壓縮測試的核心。）

## 題旨
因為（X：行為／觀念），導致（Y：結果）。
（最小可檢查形式。例：因為驕傲，導致自取滅亡。老手可違反此公式——違反時按下方規則的三項標註。）

## 結局方向
（結局是回頭修正前文的樞紐。全書結局已定則寫定；僅局部就緒可誠實標「未定」。）

## 基調（氛圍／筆調）
（一句話定調：本書說故事的態度，如「哥特恐怖＋黑色幽默」。這是 `story/設定/風格/風格.md`
 承接展開的上游源頭，也是「筆調前後一致」的基準。見 `氛圍與筆調.md`。多重基調標主從。）

## 敘事視角結構
（書級 POV 決策：單線或多線；各線 POV 角色與人稱（如「第三人稱有限／Deep POV」）；多線如何切換。
 承重創作分叉，一經拍板記於此；`write` 每章 POV 以此為權威來源。未定的部分誠實標「未定」。）

## 取向定位（爽感↔深度）＋節奏檔位
（書級「網文↔文學」定位：本書在五條對衝軸偏哪端——節奏速度／揭露反轉速度／主題比重／
 資訊密度／連載鉤子強度。可整體一句、逐軸標、或**分 arc 標**。承重分叉一經拍板記於此。
 節奏檔位（開頭段／常態段）是本項的子維度，分 arc 標定位時一併註明——見下方規則。）
```

- **源檔只寫「主線」不寫「主題」**；不在此展開細節壓縮或文筆——那是衍生檔的事。
- 承重決定未全定時可誠實標「未定」，但單線/多線、取向大方向應在下沉幕綱前定案。

```markdown

generated-at: 2026-07-19
主線: 一句話                            # 抄自源的一句話（機讀快取）
題旨: { X: 驕傲, Y: 自取滅亡 }          # 機讀；違反基本公式時見規則改標
基調: 哥特恐怖＋黑色幽默                 # 抄自源，供風格檔承接與一致性核對
基調主從: [哥特恐怖(主), 黑色幽默(從)]   # 多重基調時標
視角結構: { 線: 單線, POV: 小女巫, 人稱: 第三人稱有限·DeepPOV }
取向定位: { arc01: 偏爽, arc02: 偏文學 }  # 逐軸/整體/分arc，抄自源決定
節奏檔位: { arc01: 開頭段, arc02: 常態段 }  # 取向定位子維度
貫穿大懸念: 主角真身之謎                  # 常態段宣告靠長線供能時，可指認的那條
---
## 壓縮（同一主線的三種長度，內容一致、詳略不同）
### 50 字
### 100 字
### 200 字
## 高概念（Look / Hook / Book）
- Look：一句能看見畫面的賣點
- Hook：勾住讀者的鉤子
- Book：這是一個什麼類型／規模的故事
## 取向定位分析
（把源的取向定位決定展開成逐軸分析：五條對衝軸各偏哪端、為何、如何驅動爽點密度與節奏；
 分 arc 時逐 arc 說明檔位與其結構位置是否相稱。是 `beat-test`／`write-test`／`outline-test`
 對照的權威分析。）
## 待裁決回饋
| 日期 | 來源 | 發現 | 建議回寫（改哪個源檔） | 狀態 |
|------|------|------|----------------------|------|
```

- **壓縮三段皆須存在**：50／100／200 字是同一主線的壓縮，彼此不得矛盾，也不得與源的「一句話」漂移。
- **節奏檔位（開頭段／常態段）是取向定位的子維度、不另立欄位**：
  - **開頭段**：懸「主角命運」，高密度、事件驅動、幾乎每章一個轉折或一次變強，快速釘住讀者。
  - **常態段**：懸「這一場誰死／怎麼打臉」，靠一條**貫穿核心大懸念**在底層供能，章末鉤降級為技能解鎖／喜劇釘子。
  - 常態段若宣告「靠長線大謎供能」，那條**貫穿核心大懸念**應在本書某處可指認（front-matter `貫穿大懸念` 欄記之，`outline-test` 會查）。詳見 `技巧知識庫/戲劇張力與懸念.md` §六(d)。

## 規則

- **一句話寫不出＝主線未定**：源檔一句話缺失時，`organize`／`develop` 應在缺口診斷報告標「主線待定」。
- **題旨**必須能寫成「因為 X，導致 Y」；X 是行為或觀念、Y 是結果。填不出＝`organize` 標「題旨待定」。
- **題旨違反基本公式時**（`主題題旨.md`「老手可違反此公式」）：不得自由發揮，一律按「違反形式／為何違反／可否回頭套用基本公式」三項標註（源檔題旨區塊寫這三項，衍生 front-matter `題旨` 欄改標 `違反: <一句>`），確保不同 session 對同一例外的記錄一致、可比對。
- **基調是風格檔的上游**：`story/設定/風格/風格.md` 的 `基調參照` 承接本欄；基調要變改此源，風格檔隨之重生（見 `風格.schema.md`）。
- **敘事視角結構／取向定位是承重創作分叉**：一經拍板記於源檔，`write` POV／`beat-sheet`·`write` 爽點密度與節奏皆以此為權威，不另散落他處。允許分 arc 標不同取向。
- **源→衍生分權**：源 `00-摘要.md` 只由作者或 `organize`／`develop`（作者確認後）修改；`beat-sheet`／`beat-test`／`outline-test`／`write` 等下游 skill 只能在衍生 `00-摘要.ai.md` 的「待裁決回饋」區追加列，不得改動源或衍生的分析區。回饋由 `organize`／`develop` 回頭消化（提請作者改源後重生）。
- 重生後以 `tools/derived_sync stamp 00-摘要.ai.md` 封章（別手算 hash）。

## 檢查點（供 outline-test 參照）

- 源檔能否用一句話說清主線？（否 → 主線未收斂）
- 題旨是否為可檢查的「因為 X 導致 Y」形式？
- 基調是否明確，且與幕綱/大綱的實際調性一致（無中途轉調）？並已被 `風格.ai.md` 正確承接？
- 摘要的立場／題旨，是否與幕綱、大綱一致（無中途漂移）？
- 取向定位是否已標（或誠實標「未定」）？若已標，幕綱/正文的爽點密度與節奏，是否與宣告的定位一致（無「說要文學向卻通篇快爽套路」之類的落差）？
- 若分 arc 標了**節奏檔位**（開頭段／常態段）：宣告「常態段靠長線大謎供能」時，那條**貫穿核心大懸念**在摘要/大綱裡是否可指認（front-matter `貫穿大懸念` 是否有著落）？各 arc 檔位是否與其結構位置相稱（開頭 arc 標常態、或中段 arc 仍套開篇緊繃＝可疑）？（詳見 `outline-test`「貫穿大懸念」、`beat-test`「節奏檔位與資訊差引擎」。）
- `00-摘要.ai.md` 是否 fresh（`derived-sync check` 不報 stale）？stale＝源 `00-摘要.md` 改過未重生。
