# 結構定義：世界觀（`story/參照/世界觀.md`）

> **這是什麼**：定義「符合結構的世界觀參照」長怎樣，作為粗階段 `develop`／`organize` 產出、以及一致性提示的基準。
> **方法依據**：`技巧知識庫/世界觀與背景.md`（背景七維度、三要領、與劇情扣合）。
> ⚠️ **輕量、可改、非受保護**：本檔不套用舊計畫「bible 受保護／凍結」模型；那屬動筆後的後期基建。世界觀在動筆前應能自由修改。

---

## 用途

把散落在 `raw/` 的世界觀／設定，整理成有結構、能服務劇情的一份參照，讓作者看得出「哪些維度已定、哪些空白、哪裡可能矛盾」。世界觀是**與主線半獨立的產物軸**：它可以很成熟，而主線／題旨還很空（反之亦然）。

## 檔案格式

```markdown
# 世界觀

## 一句話定位
（這是個什麼樣的世界／設定的高概念，一句話。）

## 核心規則（力量／魔法／社會體系）
- 規則：……
  - 影響力：這條規則如何束縛或推動角色、能催生什麼衝突？（抽掉它劇情會不會垮？）
- 規則：母親臨死以愛保護主角，在他身上留下護盾。（🧊 水下｜揭示於 收[[伏筆:母愛護盾]]）
  - 影響力：……
  - （後段才向讀者揭示的規則加 🧊 標記，語法見 `共同約定.md` 四；標記本身不改變規則怎麼寫。）

## 背景維度盤點
| 維度 | 內容 | 狀態 |
|------|------|------|
| 時間 |  | 已定／待決／空白 |
| 空間 |  |  |
| 活動 |  |  |
| 地點 |  |  |
| 組織 |  |  |
| 社會 |  |  |
| 自身 |  |  |

## 待確認／潛在矛盾
- （設定衝突、自洽存疑處，逐條列出，交作者裁決——只標示，不擅自拍板。）

## 素材出處
- （本世界觀彙整自 `raw/` 的哪些檔案。）

```
（下游回饋記錄區塊格式見 `結構定義/回饋記錄.schema.md`。）

## 規則

- **核心規則盡量標「影響力」**：一條設定若抽掉後劇情照樣成立，它只是點綴；好設定應「抽掉就垮」（見 `世界觀與背景.md` 三要領）。
- **背景維度盤點**用七維度清單（時間／空間／活動／地點／組織／社會／自身），讓覆蓋與空白一目了然；空白不必硬填，標「空白」即可。
- **矛盾只標示、不代決**：潛在矛盾逐條攤開交作者裁決（AI 是審稿員，不是門檻）。
- **後段才揭示的規則加 🧊 標記**：語法與解析規則見 `共同約定.md` 四。標記只放一個指向幕綱 `收[[伏筆:x]]` 的指標，**不在此用散文複述何時揭示**——揭示時機的真實來源是幕綱。
- 本檔為**參照**，非主線；不在此寫劇情走向（那屬摘要／大綱）。
- **正文與回饋記錄分權**：本體（一句話定位／核心規則／背景維度盤點／待確認潛在矛盾）只由作者、`develop`、`worldbuild`（皆作者確認後）修改——`develop` 做粗階段冷啟動彙整，`worldbuild` 做粗階段後的橫向補厚/消矛盾（同屬設定層，非下游）；`beat-sheet`／`beat-test`／`outline-test`／`expand` 等下游 skill 只能在「下游回饋記錄」區塊追加列，不得改動正文（`worldbuild` 會回頭消化這些回饋列）。

## 檢查點（供一致性提示參照）

- 核心規則是否具「影響力」（服務劇情、不可隨意抽換）？
- 每個 🧊 標記是否解析得到對應的 `收[[伏筆:x]]`，或明標跨集留白？（見 `共同約定.md` 四；由 `beat-test` 機械核對，找不到 `收` 且該 arc 未拆時列為資訊、非可疑點）
- 七維度是否有**關鍵**空白會擋住主線收斂？（非關鍵空白可留白）
- 有無自相矛盾／自洽存疑？
  > ⚠️ 若矛盾涉及**魔法／力量體系**，可依 `世界觀與背景.md`「魔法／力量體系的自洽建構」節的 Sanderson 三定律檢查：解決衝突的魔法讀者是否理解、限制是否比能力更清楚、是否優先擴充既有規則而非亂加新規則。
  > ⚠️ **其餘領域**（社會規則、時間線、組織架構等）的設定一致性判斷，目前只能以 **LLM 審稿員判斷**代行——`Data/` 缺此方法的原文（〈下筆好設定〉遺失）。輸出為「可疑點提示」，非 pass/fail，且須標明此法非 Data 原文方法。
