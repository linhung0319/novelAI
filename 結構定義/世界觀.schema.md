# 結構定義：世界觀（`story/設定/世界觀/`）

> **這是什麼**：定義「世界觀」設定的源→衍生兩檔格式。源檔是作者自由手寫的某個設定主題（魔法／功法／組織／時間…）；衍生檔是 AI 依源檔生成的「限制與代價／影響力／自洽／升格哨兵」分析。
> **方法依據**：`技巧知識庫/世界觀與背景.md`（背景七維度、三要領、與劇情扣合；魔法/力量體系用 Sanderson 三定律）。
> **所有權**：見 `共同約定.md` 二（源→衍生）。取代舊的單一 `story/參照/世界觀.md`。

---

## 檔案佈局

```
story/設定/世界觀/
  <主題>.md         # 人管·源：一主題一檔（魔法體系/巫師組織/時間結構…；仙俠即功法/招式）
  <主題>.ai.md      # AI管·衍生：限制·代價·影響力·自洽·🧊·升格哨兵（重生自同名 .md）
  _總覽.ai.md       # AI管·rollup：七維度盤點＋核心規則索引＋待確認矛盾＋升格哨兵彙總
```

- **一主題一檔**：作者要加一條設定，就在 `世界觀/` 下新增一個 `<主題>.md`。這也讓「worldbuild 隨類型而異」（西幻魔法／仙俠功法招式）自然落地為不同主題檔，無需特例。
- 衍生檔由 `develop`／`worldbuild`（皆作者確認後）生成，作者**不手改**；改設定去改源檔、再重生。

## 源檔 `<主題>.md`（人管，無 schema）

作者自由描述這個設定主題即可——**沒有格式要求，重點是好讀好改**：

```markdown
# 芯片金手指

主角腦中一塊會隨他成長、越變越聰明的 AI 芯片。收集越多相關資料，
越能對那類東西做「優化」。他能跟芯片對話。餵給芯片的，是他用地球
科技原理去拆解巫術知識後的理解。
（想到什麼寫什麼；限制與代價可以寫、也可以留給 AI 補。）
```

## 衍生檔 `<主題>.ai.md`（AI 生成，嚴格）

```markdown
---
generated-from: <源檔 hash>       # 由 tools/derived_sync stamp 蓋，勿手填
generated-at: 2026-07-19
主題: 芯片金手指
影響力: 高                         # 抽掉劇情會不會垮：高／中／低
主從: 主打                         # 主打／佐料（元素主從，供防過載）
維度: [自身]                       # 對應七維度中的哪些（供 _總覽 彙整）
伏筆: { 埋: [], 收: [異世界門徑] }
🧊水下: false                      # 或填揭示點，如 "收[[伏筆:異世界門徑]]"
升格哨兵: false                    # true＝此主題帶硬約束，供 beat-test/sync 盯
---
## 限制與代價（Sanderson）
- 上限＝…（限制／Sanderson 第二定律）
- 代價＝…
## 影響力
抽掉會怎樣、如何服務劇情、如何避免機器神。
## 自洽 / 升格哨兵
（帶硬約束時，逐條列「三前提恆守」等，供下游盯。）
## 待裁決回饋
| 日期 | 來源 | 發現 | 建議回寫（改哪個源檔） | 狀態 |
|------|------|------|----------------------|------|
```

- **後段才揭示的規則加 🧊 標記**（語法見 `共同約定.md` 六）：只放指向幕綱 `收[[伏筆:x]]` 的指標，不散文複述揭示時機。

## 規則

- **核心規則盡量標「影響力」**：一條設定抽掉後劇情照樣成立，它只是點綴；好設定應「抽掉就垮」（見 `世界觀與背景.md` 三要領）。
- **主從標記防過載**：主打／佐料明標；佐料不得升格為承重（升格哨兵是這條的落地）。
- **維度覆蓋**看 `_總覽` 的七維度盤點；空白不必硬填，標「空白」即可。
- **矛盾只標示、不代決**：`_總覽` 的「待確認／潛在矛盾」逐條攤開交作者裁決。
- **源→衍生分權**：作者只改源 `<主題>.md`；衍生 `.ai.md`／`_總覽.ai.md` 由 `develop`／`worldbuild`（皆作者確認後）重生。下游 skill（`beat-sheet`／`write`／`expand`…）發現問題，只在對應衍生檔「待裁決回饋」區追加列，交 `worldbuild` 回頭消化（提請作者改源後重生）。
- 重生後以 `tools/derived_sync stamp <檔>.ai.md` 封章。

## 檢查點（供一致性提示參照）

- 核心規則是否具「影響力」（服務劇情、不可隨意抽換）？
- 每個 🧊 標記是否解析得到對應的 `收[[伏筆:x]]`，或明標跨集留白？（由 `beat-test` 機械核對）
- 七維度是否有**關鍵**空白會擋住主線收斂？（非關鍵空白可留白）
- 有無自相矛盾／自洽存疑？
  > ⚠️ 涉及**魔法／力量體系**：依 `世界觀與背景.md`「魔法／力量體系的自洽建構」的 Sanderson 三定律檢查（解決衝突的魔法讀者是否理解、限制是否比能力清楚、是否優先擴充既有規則）。
  > ⚠️ **其餘領域**（社會／時間線／組織）的一致性目前以 **LLM 審稿員判斷**代行（`Data/` 缺此方法原文），輸出為「可疑點提示」非 pass/fail，須標明非 Data 原文方法。
- `_總覽.ai.md` 與各 `<主題>.ai.md` 是否 fresh（`derived-sync check` 不報 stale）？
