# 結構定義：幕綱（`story/幕綱/arcNN.md`）

> **這是什麼**：定義「符合結構的幕綱」長怎樣。**這是本系統最關鍵的 schema**——幕綱是結構化 Markdown，讓因果、伏筆、結構可被機械抽取並跑測試。
> **方法依據**：`技巧知識庫/因果法則.md`（幕的五要素、前果＝後因）、`技巧知識庫/伏筆與意外.md`（伏筆閉合）、`技巧知識庫/結構公式.md`（結構階段）。

---

## 用途

幕綱＝把故事拆成**有序、離散的「幕」**，是小說的可執行藍圖。每一幕是「角色在某時某地做了某個行為，帶來衝突並產生結果」。

## 檔案配置

- 每個弧線（arc）一個檔：`story/幕綱/arc01.md`、`arc02.md`……
- 索引檔 `story/幕綱/_index.md`（見下「索引檔」節）。

## 幕號配號規則（號段預配）

- **幕號是穩定 ID，不代表全書順序。** 全書的幕排列順序由 `幕綱/_index.md` 的 arc 排列決定，不由幕號大小決定。
- **每個 arc 下沉時預配一段 100 號的號段**：arc01＝幕001–100、arc02＝幕101–200、arc03＝幕201–300……以此類推。
- 這讓 arc 可以**亂序下沉**（先想清楚 arc03 就先拆 arc03，配幕201 起的號段），不必猜中間 arc 需要幾幕、也不必事後重編號——`[[幕NNN]]`、`埋/收[[伏筆:x]]`、章節錨點 `<!-- 幕NNN -->` 全都引用幕號，重編號的代價極高，禁止重編。
- 一個 arc 超過 100 幕時（罕見），向作者確認後為該 arc 追加下一個未用號段，並於 `_index.md` 註明。
- 歷史例外：在本規則生效前產出的 arc（如 gothic_witch arc01＝幕001–007）保持原號，不回頭改。

## 單一幕的格式

每一幕是一個 `##` section，標題為 `幕NNN · 幕名`，內含**固定欄位**：

```markdown
## 幕023 · 冰原奪髓
- 角色：林小凡、血衣老者
- 時空：北域冰原／第三夜
- 行動：闖禁地取寒髓
- 衝突：老者守護、出手阻攔
- 結果：奪得寒髓，左臂遭封脈
- 前因：[[幕019]]（得知寒髓可解毒）
- 伏筆：埋[[伏筆:封脈之術]]
- 結構階段：磨練成長

（下方可續寫自由靈感註備，捕捉當初腦中的畫面、對白、運鏡等，不限格式。）
```

### 固定欄位（八項）

| 欄位 | 說明 | 對應方法 |
|------|------|----------|
| 角色 | 本幕的主角與配角 | 五要素 |
| 時空 | 時間／地點（與前幕相同可省略，但仍存在） | 五要素 |
| 行動 | 角色做的行為（廣義，含「想事情」） | 五要素 |
| 衝突 | 本幕不可缺少的衝突 | 五要素 |
| 結果 | 成功／失敗／有成有敗／暫時保留；須是「能催發下一幕」的有意義結果 | 五要素、前果＝後因 |
| 前因 | 用 `[[幕NNN]]` 指向作為本幕動機來源的前一幕 | 前果＝後因 |
| 伏筆 | 用 `埋[[伏筆:x]]` / `收[[伏筆:x]]` 標記；本幕無則留「—」 | 伏筆閉合 |
| 結構階段 | 本幕對映選用公式的哪個階段（如「磨練成長」） | 結構公式 |

## 連結規範（機械可抽取）

- **`[[幕NNN]]`**：指向作為前因的幕。NNN 為三位數字（如 `[[幕019]]`）。
- **`埋[[伏筆:名稱]]`**：伏筆的埋設點。
- **`收[[伏筆:名稱]]`**：伏筆的回收點。名稱須與對應的「埋」完全一致。

### 單一真實來源（重要）

**伏筆不另立登記簿。** 幕綱裡的 `埋`／`收` 標記就是唯一真實來源；某伏筆是否閉合，直接從幕綱掃描 `埋[[伏筆:x]]` 與 `收[[伏筆:x]]` 算出。若需要伏筆總覽，由 AI 從幕綱生成**視圖**，作者永遠只維護幕綱這一份。（刻意避開「多層登記簿要記得同步」的壞味道。）

## 本 arc 伏筆狀態（選用，跨 arc／全書懸念時使用）

當某條伏筆的埋設點與回收點**不在同一個 arc**（落在尚未下沉的 arc、全書終局，或永久留白），不必為了湊對而在本 arc 內硬寫另一半。在該 arc 檔案結尾加一個「本 arc 伏筆狀態」表，誠實記錄兩端各在哪。

arc 亂序下沉時**兩個方向都會發生**，兩種都用同一張表：

```markdown
## 本 arc 伏筆狀態
| 伏筆 | 埋設幕 | 收回 | 備註 |
|------|--------|------|------|
| 起火原因 | 幕004 | 未收（刻意留白，見 世界觀.md 待確認節） | 貫穿全書的懸念鉤子 |
| 母愛護盾 | arc01（未拆） | 幕409 | 收在本 arc、埋設點在尚未拆幕的 arc01；埋設屬主角倖存之謎 |
```

第二列是**反向列**——本 arc 只收，埋設點落在還沒拆幕的 arc。等 arc01 拆出幕號，把「埋設幕」欄改填實際幕號（`幕010`）即可；這張表就是回補的依據。

### 每一欄裝什麼

| 欄 | 內容 |
|----|------|
| 伏筆 | 伏筆名，與幕裡的 `埋`／`收[[伏筆:x]]` 完全一致 |
| 埋設幕 | 埋設點的 `幕NNN`；埋設點所在的 arc 尚未拆幕時填 `arcNN（未拆）` |
| 收回 | 回收點的 `幕NNN`；本書內不收時填「未收（原因）」；回收點所在的 arc 尚未拆幕時填 `arcNN（未拆）` |
| 備註 | 散文，寫這條伏筆的**作用與理由**——懸念功能、與世界觀規則的對應、刻意留白的緣由 |

**`埋[[伏筆:x]]`／`收[[伏筆:x]]` 標記只存在於幕的「伏筆」欄。** 這張表是那些標記的**視圖**（見上「單一真實來源」），不是它們的第二個家：表格四欄（含備註散文）裡提到伏筆時，一律只寫伏筆名的純文字。在備註裡寫出字面標記，機械掃描會把它當成一個真的埋設點，於是把一條根本還沒埋的伏筆判成已閉合。

### `beat-test` 怎麼讀這張表

- 表內列出的跨 arc／留白伏筆 → **資訊**，不是可疑點；表格外的「埋而未收／收而未埋」才是可疑點。
- **`arcNN（未拆）` 是有條件的**：它以 `幕綱/_index.md` 裡該 arc 仍標未拆為前提。該 arc 已拆出幕號、而這一端仍未回填實際幕號 → **可疑點**（跨 arc 待補已到期）。

## 索引檔 `story/幕綱/_index.md`

作為 AI 檢索入口，概覽各 arc：

```markdown
# 幕綱索引

- 選用結構公式：編劇九階段
- 全書順序：arc01 → arc02 → …（幕的先後以此排列為準，非幕號大小）
- arc01：幕001–幕012（號段 001–100）　起（平凡失衡→召喚衝突）
- arc02：幕101–幕116（號段 101–200）　承（推動冒險→磨練成長）
- …
```

## 檢查點（供 beat-test 參照）

- **因果連續性**：每幕「結果」是否催發下一幕動機？`[[幕NNN]]` 前因鏈是否完整？有無「有因無果／有果無因」的孤兒、或未連主線的幕？
- **伏筆閉合**：每個 `埋[[伏筆:x]]` 是否有對應 `收[[伏筆:x]]`？結尾有無懸空的埋而未收？（已在「本 arc 伏筆狀態」表中誠實標注跨 arc／全書懸念者除外——那是資訊，不算懸空；但表中標 `arcNN（未拆）` 的那一端，若 `_index.md` 顯示該 arc 已拆，即為到期未回補的可疑點）
- **結構完整性**：各幕的「結構階段」對照 `參照/結構.md` 選用的公式，有無缺階段／重複／錯序？
