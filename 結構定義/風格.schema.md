# 結構定義：風格（`story/設定/風格/`）

> **這是什麼**：定義「風格（腔調／語域／筆調執行層）」設定的源→衍生兩檔格式。源檔是作者自由手寫的「這本書該用什麼腔調寫」；衍生檔是 AI 依源檔生成的「可被 `write-test` 逐條核對的機器事實（禁用詞表／稱謂系統／可用句式）＋腔調・時代文化色的 prose 分析」。這是一個**新產物**——把過去散在 `00-摘要.md`「基調」與各技法檔裡的「筆調執行約束」收成一份可機讀的權威檔。
> **方法依據**：`技巧知識庫/氛圍與筆調.md`（基調／筆調前後一致）、`技巧知識庫/字詞句法與抗生硬.md`（M7 語域錯配／時代錯亂詞彙＝抗生硬普世地板）、`技巧知識庫/修辭與字句鍛鍊.md`（文學向修辭天花板，依取向定位選用）。
> **所有權**：見 `共同約定.md` 二（源→衍生）。**與 `00-摘要.md`「基調」的分工**：摘要「基調」是**書級一句話定調**（哥特恐怖＋黑色幽默）、由作者在源 `00-摘要.md` 拍板；本檔是那句定調的**執行細則落地**（哪些詞禁用、人物怎麼稱呼、句式偏好），供 `write`／`revise` 落筆、`write-test` 核對。基調變更的真實來源仍是摘要；風格檔承接它並展開為可核對事實。

---

## 檔案佈局

```
story/設定/風格/
  風格.md          # 人管·源：作者自由寫「這本書該用什麼腔調」，無格式要求
  風格.ai.md       # AI管·衍生：禁用詞表/稱謂系統/可用句式(front-matter)＋腔調·時代文化色(本體)
```

- **全書單一檔起步**：多數書一份 `風格.md` 即足。
- **forward-compatible 可日後拆多檔**：多世界並存時可拆 `風格/主世界.md`、`風格/異界.md`（各自成源→衍生對；rollup 規則比照 `世界觀/`，同層新增 `_總覽.ai.md`）。章節在 `chNN.ai.md` front-matter 以選填 `風格：<檔名>` 指定該章套哪一份（見 `章節.schema.md`）；單檔時該欄可省略，預設 `風格.ai.md`。
- 衍生檔由 `develop`（作者確認後）生成，作者**不手改**；改腔調去改源 `風格.md`、再重生。

## 源檔 `風格.md`（人管，無 schema）

作者自由描述這本書想要的腔調即可——**沒有格式要求，重點是好讀好改**。以下只是「可以寫什麼」的靈感，不是欄位：

```markdown
# 風格

哥特恐怖打底、混一點黑色幽默。維多利亞時代感，敘述端莊、句子偏長，
但恐怖高潮要能切成短促的碎句。人物之間用「先生／小姐／閣下」這類
稱呼，不要現代口語（別出現「老周」「兄弟」「OK」這種）。譬喻可以
華麗一點，但別濫情。
（想到什麼寫什麼；禁用詞、稱謂、句式偏好可以寫、也可以留給 AI 補。）
```

## 衍生檔 `風格.ai.md`（AI 生成，嚴格）

```markdown
---
generated-from: <源檔 hash>       # 由 tools/derived_sync stamp 蓋，勿手填
generated-at: 2026-07-19
風格: 主世界                        # 選填：多檔時填此檔的世界/線名；單檔可省略
基調參照: 哥特恐怖＋黑色幽默         # 引 00-摘要.md「基調」（不重新定義，只承接）
禁用詞表: [老周, 兄弟, OK, 搞定, 給力]   # write-test M7 語域/時代錯亂逐條核對用
稱謂系統: { 敬稱: [先生, 小姐, 閣下], 禁: [老X, 兄弟] }   # 人物互稱規範
可用句式: { 常態: 端莊長句, 恐怖高潮: 短促碎句 }          # 依段落性質切換的句式檔位
語域: 書面·古典                    # 書面/口語·古典/現代（供 M7 語域錯配核對）
---
## 腔調（說故事的態度）
（承接基調，展開成落筆時的具體態度：端莊到什麼程度、幽默怎麼放、恐怖怎麼收。）
## 時代・文化色
（時代錯亂詞彙的判準來源：這是什麼時代／文化圈的世界，哪些詞會破壞氛圍。
 供 write-test 判「時代錯亂詞彙」——禁用詞表是它的機讀清單，本節是判準理由。）
## 句式與節奏偏好
（常態段 vs 高潮段的句長／碎句切換規則；與 `畫面感與電影化.md` 句長＝鏡頭停滯呼應。）
## 修辭天花板（依取向定位選用）
（文學向才展開：譬喻風格、擬詩手法的偏好與禁忌；純網文向可標「以抗生硬地板為準，不追天花板」。）
## 待裁決回饋
| 日期 | 來源 | 發現 | 建議回寫（改哪個源檔） | 狀態 |
|------|------|------|----------------------|------|
```

- **front-matter 三欄是 `write-test` M7／測試10 的機讀核對基準**：`禁用詞表`（掃到即候選命中）、`稱謂系統`（人物互稱是否越界）、`可用句式`（段落性質對不對檔位）。哪些詞入表由 AI 依源檔＋時代文化色推導，作者確認後封章。
- **禁用詞表是「機讀清單」、時代・文化色是「判準理由」**：兩者一體——表給 `write-test` 快掃，理由供作者裁決與 AI 推導新詞；勿只填表不寫理由（下次重生無從擴充）。

## rollup（多檔時才有）`_總覽.ai.md`

單檔起步時**無 rollup**。日後拆多世界檔，才在 `story/設定/風格/` 下補 `_總覽.ai.md`，格式比照 `世界觀.schema.md` 的 rollup 精神：列各風格檔的一句話定位＋適用世界/線＋禁用詞表交集/差異，供 `write` 選章風格時對照。源＝同層所有 `*.md` 的集合（digest 規則見 `共同約定.md` 三）。

## 規則

- **基調不在此重新定義**：`基調參照` 欄只引 `00-摘要.md`「基調」，不另立第二份定義（避免與摘要漂移）。基調要變，改源 `00-摘要.md`，風格檔隨之重生承接。
- **禁用詞表服務「語域錯配／時代錯亂」這條普世地板**（`字詞句法與抗生硬.md` M7 病徵10、`章節.schema.md`「流暢四層次」第③層），**與取向定位無關**——網文向一樣要防「哥特世界冒出『老周』」。
- **修辭天花板才綁取向定位**：文學向展開，純網文向可標「不追天花板」；`write-test` 對修辭的檢查只在文學向段落重點跑（見 `修辭與字句鍛鍊.md`）。
- **源→衍生分權**：作者只改源 `風格.md`；衍生 `.ai.md` 由 `develop`（作者確認後）重生。下游 skill（`write`／`revise`／`write-test`）發現問題，只在衍生檔「待裁決回饋」區追加列，交 `develop` 回頭消化（提請作者改源後重生）。
- 重生後以 `tools/derived_sync stamp 風格.ai.md` 封章（別手算 hash）。

## 檢查點（供 write-test 參照）

- 本章實際用詞是否命中 `風格.ai.md` 的 `禁用詞表`？（命中＝語域/時代錯亂候選，交 write-test 測試10 複核）
- 人物互稱是否符合 `稱謂系統`（無現代口語越界）？
- 段落性質（常態／恐怖高潮／動作…）是否套了對應 `可用句式` 檔位，還是全篇一種句長勻平？
- 本章筆調是否與 `基調參照`（＝`00-摘要.md` 基調）一致、無不自覺轉調？（此條與 `氛圍與筆調.md` 的基調一致性檢查同源。）
- `風格.ai.md` 是否 fresh（`derived-sync check` 不報 stale）？stale＝源 `風格.md` 改過未重生。
